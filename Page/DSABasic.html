📘 Data Structures and Algorithms (DSA) – Simple Notes

These notes are written in easy language, interview‑friendly, and good for quick revision.

🔹 What is DSA?

Data Structures and Algorithms (DSA) is the study of organizing data and solving problems efficiently using logical steps.

🔹 What is a Data Structure?

A data structure is a way to store and organize data so that it can be used efficiently.

Types of Data Structures

Data structures are used to store and organize data efficiently.

1. Primitive Data Structures

Store single values.

  • int – integer values
  • float / double – decimal values
  • char – single character
  • boolean – true / false
2. Non-Primitive Data Structures

Store multiple or complex data.

A. Linear Data Structures

Elements are stored in a sequence.

  • Array – fixed size, indexed
  • Linked List – nodes connected using pointers
  • Stack – LIFO (Last In First Out)
  • Queue – FIFO (First In First Out)
  • Deque – insertion/deletion from both ends
B. Non-Linear Data Structures

Elements are not stored sequentially.

  • Tree – hierarchical structure
  • Graph – nodes connected by edges
  • Hash Table – key–value pairs

🔹 What is an Algorithm?

An algorithm is a step‑by‑step procedure to solve a problem.

Example: Steps to find the largest number in an array.

📦 Data Structures (Easy Definitions)

1️⃣ Array

An array is a collection of elements stored in contiguous memory locations and accessed using an index.

Key points:

  • Fixed size (decided at creation)
  • Stores elements of same data type
  • Fast access using index (O(1)) or constant time complexity.
  • Insertion/deletion is costly due to shifting

Interview use: Used when size is known and fast access is required.

2️⃣ Linked List

A linked list is a linear data structure where elements are stored as nodes connected using pointers.

Key points:

  • Dynamic size
  • Non-contiguous memory
  • Each node contains data + next pointer
  • Insertion/deletion is efficient
  • Slower access compared to array (O(n))

Interview use: Used when frequent insertions/deletions are needed.

3️⃣ Stack

A stack is a linear data structure that follows the LIFO (Last In First Out) principle.

Key points:

  • Operations: push, pop, peek
  • Access from one end only (top)
  • Overflow and underflow conditions

Example: Plate stack

Interview use: Function calls, undo/redo, expression evaluation.

4️⃣ Queue

A queue is a linear data structure that follows the FIFO (First In First Out) principle.

Key points:

  • Operations: enqueue, dequeue
  • Insertion from rear, deletion from front
  • Can be implemented using array or linked list

Example: Line at a ticket counter

Interview use: Scheduling, buffering, task processing.

5️⃣ Deque

A deque is a double-ended queue where insertion and deletion can occur from both ends.

Key points:

  • Can act as stack and queue
  • Types: input-restricted, output-restricted
  • Flexible data handling

6️⃣ Hashing

Hashing is a technique to map data to a fixed size using a hash function for fast access.

Key points:

  • Uses key-value pairs
  • Average time complexity O(1)
  • Collision handling using chaining or open addressing

Interview use: Databases, caches, symbol tables.

7️⃣ Tree

A tree is a non-linear data structure consisting of nodes connected by edges, with a hierarchical structure.

Key points:

  • Has root, parent, child, leaf
  • No cycles
  • Used to represent hierarchy

Interview use: File systems, XML/JSON parsing.

8️⃣ Binary Tree

A binary tree is a tree where each node has at most two children.

Key points:

  • Children are left and right
  • Types: full, complete, skewed
  • Traversals: inorder, preorder, postorder

9️⃣ Binary Search Tree (BST)

A BST is a binary tree where the left child is smaller and the right child is greater than the root.

Key points:

  • Inorder traversal gives sorted order
  • Average search time O(log n)
  • Worst case O(n) if skewed

Interview use: Searching and sorting applications.

🔟 Heap

A heap is a complete binary tree used to find minimum or maximum elements efficiently.

Key points:

  • Types: Min Heap, Max Heap
  • Root element is min or max
  • Used in priority queues

Interview use: Scheduling, shortest path algorithms.

1️⃣1️⃣ Graph

A graph is a collection of nodes (vertices) connected by edges.

Key points:

  • Directed or undirected
  • Weighted or unweighted
  • Can contain cycles
  • Traversals: BFS, DFS

Interview use: Social networks, maps, network routing.

⚙️ Algorithms (Easy Definitions)

1️⃣ Linear Search

Linear search checks each element one by one to find the target value.

2️⃣ Binary Search

Binary search repeatedly divides a sorted array into halves to find the target.

3️⃣ Sorting

Sorting is the process of arranging elements in ascending or descending order.

Common Sorting Algorithms:

  • Bubble Sort
  • Selection Sort
  • Insertion Sort
  • Merge Sort
  • Quick Sort

4️⃣ Recursion

Recursion is a technique where a function calls itself to solve a problem.

5️⃣ Greedy Algorithm

A greedy algorithm makes the best choice at each step to reach an optimal solution.

6️⃣ Dynamic Programming (DP)

Dynamic programming solves problems by breaking them into overlapping subproblems and storing results.

7️⃣ Backtracking

Backtracking tries all possible solutions and removes those that fail.

⏱️ Time & Space Complexity

Time Complexity

Time complexity tells how the execution time of an algorithm grows with input size.

Common notations:

  • O(1) – Constant
  • O(n) – Linear
  • O(n²) – Quadratic
  • O(log n) – Logarithmic

Space Complexity

Space complexity tells how much extra memory an algorithm uses.
