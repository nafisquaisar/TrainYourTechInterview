🎨 Android User Interface (UI)

The Android User Interface (UI) defines how an app looks and how users interact with it.
UI elements are usually designed using XML layout files.

1. Layouts

Layouts are containers that hold and arrange UI components on the screen.

👉 Think of layouts as frames that arrange buttons, text, and images.

1.1 Linear Layout

  • Arranges UI elements in a single direction

  • Can be:

    • Horizontal
    • Vertical

Use case:

  • Simple forms
  • Login screens

✔ Easy to use
❌ Not suitable for complex screens

Example Code:

<LinearLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <TextView
        android:text="Welcome" />

    <Button
        android:text="Login" />

</LinearLayout>

1.2 Relative Layout

  • Positions UI elements relative to other elements or parent

  • Example:

    • Button below text
    • Image to the right of text

✔ Flexible
✔ Reduces nested layouts
❌ Hard to manage in large designs

Example Code:

<RelativeLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <TextView
        android:id="@+id/txt"
        android:text="Hello" />

    <Button
        android:layout_below="@id/txt"
        android:text="Click Me" />

</RelativeLayout>

1.3 Constraint Layout

  • Most powerful and recommended layout
  • Positions views using constraints
  • Keeps UI responsive for all screen sizes

✔ Best performance
✔ Supports complex designs
✔ Modern standard layout

👉 Preferred layout in modern Android apps

Example Code:

<androidx.constraintlayout.widget.ConstraintLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <Button
        android:text="Submit"
        app:layout_constraintTop_toTopOf="parent"
        app:layout_constraintStart_toStartOf="parent" />

</androidx.constraintlayout.widget.ConstraintLayout>

1.4 Frame Layout

  • Designed to display single child view
  • Multiple views overlap if added

Use case:

  • Loading screens
  • Fragments container
  • Image overlays

Example Code:

<FrameLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <ImageView
        android:src="@drawable/sample" />

</FrameLayout>

1.5 Table Layout

  • Arranges views in rows and columns
  • Works like a table

Use case:

  • Form-style layouts
  • Structured input screens

❌ Less used in modern apps

Example Code:

<TableLayout
    android:layout_width="wrap_content"
    android:layout_height="wrap_content">

    <TableRow>
        <TextView android:text="Name" />
        <EditText />
    </TableRow>

</TableLayout>

1.6 Grid Layout

  • Arranges views in grid format
  • Rows and columns based layout

Use case:

  • Calculator layout
  • Gallery thumbnails

Example Code:

<GridLayout
    android:columnCount="2">

    <Button android:text="1" />
    <Button android:text="2" />

</GridLayout>

2. UI Components (Widgets)

UI components are interactive elements that users can see and touch.

2.1 TextView

  • Displays text on the screen
  • Used for labels, headings, messages

Example Code:

<TextView
    android:text="Hello Android"
    android:textSize="18sp" />

2.2 Button

  • Clickable component
  • Used to perform actions like submit, login

Example Code:

<Button
    android:text="Submit" />

2.3 ImageView

  • Displays images
  • Supports icons, photos, and graphics

Example Code:

<ImageView
    android:src="@drawable/logo" />

2.4 ImageButton

  • Button with an image instead of text
  • Used for icon-based actions

Example Code:

<ImageButton
    android:src="@drawable/icon" />

2.5 CheckBox

  • Allows multiple selections
  • Used in preference or settings screens

Example Code:

<CheckBox
    android:text="I agree" />

2.6 RadioButton

  • Allows single selection
  • Works inside a RadioGroup

2.7 RadioGroup

  • Groups RadioButtons
  • Ensures only one option is selected

Example Code:

<RadioGroup>
    <RadioButton android:text="Male" />
    <RadioButton android:text="Female" />
</RadioGroup>

2.8 Toggle Button

  • Two states: ON / OFF
  • Used for enabling or disabling features

Example Code:

<ToggleButton
    android:textOn="ON"
    android:textOff="OFF" />

2.9 SeekBar

  • Slider used to select a value

  • Example:

    • Volume control
    • Brightness control

Example Code:

<SeekBar
    android:max="100" />

2.10 RatingBar

  • Displays rating using stars
  • Used in review and feedback screens

Example Code:

<RatingBar
    android:numStars="5" />

3. Text Fields (Input Types)

Text fields allow users to enter data.

3.1 Plain Text

  • Accepts normal text
  • Used for names and general input

Example Code:

<EditText
    android:hint="Enter name"
    android:inputType="text" />

3.2 Password

  • Hides entered characters
  • Used for passwords and PINs

Example Code:

<EditText
    android:hint="Password"
    android:inputType="textPassword" />

3.3 Email

  • Accepts email format
  • Shows email-optimized keyboard

Example Code:

<EditText
    android:hint="Email"
    android:inputType="textEmailAddress" />

3.4 Phone

  • Accepts phone numbers
  • Shows numeric keypad

Example Code:

<EditText
    android:hint="Phone"
    android:inputType="phone" />

4. One-Line Memory Summary

  • Layouts → Arrange UI elements
  • UI Components → Interactive widgets
  • Text Fields → User input areas

5. Exam-Friendly Conclusion

Android UI uses layouts to structure the screen and UI components to interact with users.
Proper UI design improves user experience and app usability.
