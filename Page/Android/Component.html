🧩 Android Components – Quick Notes

Android Components are the main building blocks of an Android app.
They define how an app behaves, communicates, runs in background, and shares data.

🧱 1. Activity

An Activity represents a single screen in an Android application.

📌 Examples

  • Login screen
  • Home screen
  • Profile screen

🔄 Activity Lifecycle

The activity lifecycle describes different states of an activity from start to end.

Lifecycle Flow

  1. onCreate()
  2. onStart()
  3. onResume()
  4. onPause()
  5. onStop()
  6. onDestroy()

⚙️ Lifecycle Methods (Simple Meaning)

Method Meaning
onCreate() Activity is created
onStart() Activity becomes visible
onResume() User can interact
onPause() Activity partially hidden
onStop() Activity not visible
onDestroy() Activity destroyed

🧪 Basic Activity Code 

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
    }

}

⚙️ 2. Intents

Intents are messages used to request an action from another component.

📌 Used for

  • Opening another activity
  • Sharing data
  • Opening external apps (browser, dialer)

🎯 Explicit Intent

Used to open a specific component inside the same app.

val intent = Intent(this, SecondActivity::class.java)
startActivity(intent)

🔗 Implicit Intent

Used when the target component is not specified.
System chooses the best app.

val intent = Intent(Intent.ACTION_VIEW)
intent.data = Uri.parse("https://www.google.com")
startActivity(intent)

📦 Passing Data Using Intents

Send Data

val intent = Intent(this, SecondActivity::class.java)
intent.putExtra("username", "Nafis")
startActivity(intent)

Receive Data

val name = intent.getStringExtra("username")

⚙️ 3. Services

A Service runs tasks in the background without UI.

📌 Used for

  • Playing music
  • Downloading files
  • Syncing data

▶️ Started Service

  • Started using startService()
  • Runs independently
  • Stops manually
val intent = Intent(this, MyService::class.java)
startService(intent)

🔗 Bound Service

  • Allows components to bind and interact with the service
  • Used when continuous communication is needed

(Concept based)

🚨 Foreground Service (Concept)

  • Runs with a visible notification
  • User knows the service is running

📌 Examples

  • Music player
  • Navigation apps
  • Fitness tracking

📡 4. Broadcast Receivers

Broadcast Receivers listen for events from the system or app.

📌 Examples

  • Battery low
  • Network change
  • Phone reboot

📢 System Broadcasts

Sent automatically by Android OS.

<receiver android:name=".MyReceiver">
    <intent-filter>
        <action android:name="android.intent.action.BOOT_COMPLETED"/>
    </intent-filter>
</receiver>

📨 Custom Broadcasts

Send Broadcast

val intent = Intent("MY_CUSTOM_ACTION")
sendBroadcast(intent)

Receive Broadcast

class MyReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context?, intent: Intent?) {
        // Handle broadcast
    }
}

🗂️ 5. Content Providers

Content Providers are used to share data between different apps safely.

📌 Examples

  • Contacts
  • Call logs
  • Media files

🎯 Purpose

  • Secure data sharing
  • Controlled access
  • Standard interface for data

🧠 Use Cases

  • Reading contacts (WhatsApp, Telegram)
  • Accessing media files
  • Sharing app data with other apps

🧪 Example: Reading Contacts 

val cursor = contentResolver.query(
    ContactsContract.CommonDataKinds.Phone.CONTENT_URI,
    arrayOf(
        ContactsContract.CommonDataKinds.Phone.DISPLAY_NAME,
        ContactsContract.CommonDataKinds.Phone.NUMBER
    ),
    null, null, null
)

while (cursor?.moveToNext() == true) {
    val name = cursor.getString(0)
    val number = cursor.getString(1)
    Log.d("Contact", "$name: $number")
}

cursor?.close()

⚙️ 1. Intents

Intents are messages used to request an action from another app component.

  • Types of Intents:

    • Explicit Intent 🎯

      • Used to start a specific component (e.g., a particular Activity or Service).
      • Example: Navigating from one activity to another within the same app.

    • Implicit Intent 🔗

      • Used when you don’t specify the target component.
      • System decides which app or component can handle the request.
      • Example: Sharing content or opening a webpage.

  • Passing Data Using Intents 📦

    • You can attach data to intents using extras.

    • Example:

      val intent = Intent(this, SecondActivity::class.java)
intent.putExtra("key", "value")
startActivity(intent)

    • Retrieve data using intent.getStringExtra("key").

📡 2. Broadcast Receivers

  • Purpose: Responds to system-wide or app-wide events.
  • Example Scenario: Detecting network connectivity changes.
class NetworkChangeReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context?, intent: Intent?) {
        val connectivityManager = context?.getSystemService(Context.CONNECTIVITY_SERVICE) as ConnectivityManager
        val networkInfo = connectivityManager.activeNetworkInfo

        if (networkInfo != null && networkInfo.isConnected) {
            Toast.makeText(context, "Internet Connected", Toast.LENGTH_SHORT).show()
        } else {
            Toast.makeText(context, "Internet Disconnected", Toast.LENGTH_SHORT).show()
        }
    }
}

Register in Manifest (Static):

<receiver android:name=".NetworkChangeReceiver">
    <intent-filter>
        <action android:name="android.net.conn.CONNECTIVITY_CHANGE" />
    </intent-filter>
</receiver>

Or Register Dynamically:

val receiver = NetworkChangeReceiver()
registerReceiver(receiver, IntentFilter(ConnectivityManager.CONNECTIVITY_ACTION))

⚙️ 3. Services

  • Purpose: Perform background tasks without a UI.
  • Example Scenario: Playing music in the background.
class MusicService : Service() {
    private lateinit var mediaPlayer: MediaPlayer

    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        mediaPlayer = MediaPlayer.create(this, R.raw.song)
        mediaPlayer.start()
        return START_STICKY
    }

    override fun onDestroy() {
        mediaPlayer.stop()
        mediaPlayer.release()
        super.onDestroy()
    }

    override fun onBind(intent: Intent?): IBinder? = null
}

Start the Service:

val intent = Intent(this, MusicService::class.java)
startService(intent)

  • Purpose: Share and manage app data between different apps.
  • Example Scenario: Accessing contacts from the phone.

val cursor = contentResolver.query(

while (cursor?.moveToNext() == true) {
    val name = cursor.getString(0)

Use Case:

  • Apps like WhatsApp or Telegram use content providers to read contact data from the phone and display which contacts are available on the platform.
