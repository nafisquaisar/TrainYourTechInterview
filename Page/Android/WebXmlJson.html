🌐 Web & Networking, XML & JSON – Complete Proper Notes

This chapter explains how Android apps connect to the internet, exchange data, and use XML & JSON for data representation and UI design. These concepts are very important for real apps, exams, and interviews.

🌍 1. Web & Networking (Overview)

Web & Networking in Android means communicating with servers over the internet to:

  • Fetch data
  • Send data
  • Display web content

Most modern apps depend on internet-based services.

Examples:

  • Social media apps
  • Weather apps
  • E-commerce apps

🌐 2. WebView

WebView is a component used to display web pages inside an Android app.

📌 Why WebView?

  • Shows websites without opening a browser
  • Used for terms & conditions
  • Used for help pages

🧠 Key Points

  • Uses system web engine
  • Supports HTML, CSS, JavaScript
  • Requires internet permission

🧪 Basic Example 

val webView = findViewById<WebView>(R.id.webView)
webView.loadUrl("https://www.google.com")

🔌 3. Web Services

A Web Service is a software service that allows communication between two applications over the internet.

🧠 Simple Meaning

  • App sends request
  • Server sends response

📌 Types of Web Services

  • SOAP (old)
  • REST (modern and widely used)

🔄 4. REST API Basics

REST API stands for Representational State Transfer Application Programming Interface.
It is the most common way apps communicate with servers.

📌 Key Characteristics

  • Uses HTTP protocol
  • Lightweight
  • Stateless

🧠 Common HTTP Methods

Method Purpose
GET Fetch data
POST Send data
PUT Update data
DELETE Remove data

📱 Example

  • GET user list
  • POST login data

📦 5. Data Formats

Servers return data in specific formats so apps can understand it.

Most common formats:

  • JSON
  • XML

📘 6. XML (Extensible Markup Language)

🔹 Introduction to XML

XML is a markup language used to store and transport data in a structured format.

🧠 Why XML?

  • Human readable
  • Platform independent
  • Used widely in Android UI

🔹 XML Syntax Rules

Basic rules of XML:

  • Tags must be closed
  • Tags are case-sensitive
  • Proper nesting is required

🧪 Example 

<user>
    <name>Nafis</name>
    <age>22</age>
</user>

🔹 XML in Android Layouts

Android UI layouts are written in XML.

📌 Why XML for UI?

  • Separates UI from logic
  • Easy to design and maintain

🧪 Example 

<LinearLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <TextView
        android:text="Hello Android" />

</LinearLayout>

📗 7. JSON (JavaScript Object Notation)

🔹 Introduction to JSON

JSON is a lightweight data format used to exchange data between server and app.

🧠 Why JSON?

  • Lightweight
  • Easy to parse
  • Widely used in REST APIs

🔹 JSON Structure

JSON is written using:

  • Key–value pairs
  • Objects
  • Arrays

🧪 Example 

{
  "name": "Nafis",
  "age": 22,
  "skills": ["Android", "Kotlin"]
}

⚔️ 8. JSON vs XML

Feature JSON XML
Readability Easy Verbose
Data Size Small Large
Parsing Fast Slow
Usage APIs UI layouts

📝 Exam Tips

  • Always mention REST API uses JSON
  • XML is mainly used for Android UI
  • JSON preferred for networking

✅ Final Summary

  • WebView → Display web pages
  • REST API → Server communication
  • XML → UI & structured data
  • JSON → Fast data exchange
