Scenario-Based OOPM Questions with Answers (Java)

1️⃣ Encapsulation – Salary Control

Question:
An Employee class has salary information. Only HR should modify the salary, but others can view it. How will you design this using OOP?

Answer:
I will use encapsulation by making the salary variable private and providing public getter and setter methods to control access.

class Employee {
    private double salary;

    public double getSalary() {
        return salary;
    }

   void setSalary(double salary, String role) {
        if (role.equals("HR")) {
            this.salary = salary;
        } else {
            System.out.println("Access denied: Only HR can modify salary");
        }
    }
}

Question:
You want to protect class variables from direct access while still allowing controlled modification. Which OOP concept will you use?

I will use encapsulation by keeping variables private and exposing them through getter and setter methods.

class User {

3️⃣ Inheritance – Vehicle Example

Question:
You have a base class Vehicle and child classes Car and Bike with common properties. Which OOP concept fits here?

Answer:
Inheritance is used to share common properties and methods between related classes.

class Vehicle {
    int speed;
}

class Car extends Vehicle {
    void drive() {
        System.out.println("Car is driving");
    }
}

class Bike extends Vehicle {
    void ride() {
        System.out.println("Bike is riding");
    }
}

4️⃣ Polymorphism – Animal Sound

Question:
All animals have a sound() method, but each animal makes a different sound. How will you implement this?

Answer:
I will use inheritance and method overriding to achieve runtime polymorphism.

class Animal {
    void sound() {
        System.out.println("Animal makes sound");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Dog barks");
    }
}

5️⃣ Runtime Polymorphism – Salary Calculation

Question:
A method calculateSalary() behaves differently for full-time and part-time employees. What concept is used?

Answer:
Runtime polymorphism is used because method behavior depends on the object type at runtime.

class Employee {
    int calculateSalary() {
        return 0;
    }
}

class FullTime extends Employee {
    int calculateSalary() {
        return 50000;
    }
}

class PartTime extends Employee {
    int calculateSalary() {
        return 20000;
    }
}

6️⃣ Compile-Time Polymorphism – Method Overloading

Question:
You want multiple add() methods with different parameters. Which concept will you use?

Answer:
Compile-time polymorphism is achieved using method overloading.

class MathOperation {
    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}

7️⃣ Abstraction – ATM Example

Question:
You want to define what an ATM can do without exposing how it works internally. What will you use?

Answer:
I will use abstraction to hide implementation details and expose only essential operations.

abstract class ATM {
    abstract void withdraw();
    abstract void deposit();
}

8️⃣ Interface – Payment Methods

Question:
Different payment methods must implement a pay() method. What design will you use?

Answer:
I will use an interface to ensure all payment methods implement the same functionality.

interface Payment {
    void pay();
}

class UPI implements Payment {
    public void pay() {
        System.out.println("UPI Payment");
    }
}

class Card implements Payment {
    public void pay() {
        System.out.println("Card Payment");
    }
}

9️⃣ Abstract Class – Common Code and Mandatory Methods

Question:
You need some common code and also want to force subclasses to implement certain methods. What will you use?

Answer:
I will use an abstract class to share common code and enforce method implementation.

abstract class Shape {
    void display() {
        System.out.println("Shape");
    }

    abstract void draw();
}

🔟 Multiple Inheritance in Java

Question:
How can you achieve multiple inheritance behavior in Java?

Answer:
Multiple inheritance in Java is achieved using interfaces.

interface A {
    void show();
}

interface B {
    void display();
}

class C implements A, B {
    public void show() {}
    public void display() {}
}

1️⃣1️⃣ Final Keyword – Prevent Inheritance

Question:
You want to prevent a class from being inherited. How will you do it?

Answer:
I will mark the class as final to prevent inheritance.

final class Security {

Question:
You want to prevent a method from being overridden. What will you use?

Answer:
I will mark the method as final to prevent overriding in subclasses.

class Parent {
    final void show() {
        System.out.println("Final Method");
    }
}

1️⃣3️⃣ Parameterized Constructor

Question:
You want mandatory values during object creation. Which constructor will you use?

Answer:
I will use a parameterized constructor to enforce required values.

class Student {
    int id;
    String name;

    Student(int id, String name) {
        this.id = id;
        this.name = name;
    }
}

1️⃣4️⃣ Exception Handling

Question:
Your program may crash due to invalid input. How will you handle this?

Answer:
I will use try-catch blocks to handle exceptions and prevent application crashes.

try {
    int a = 10 / 0;
} catch (Exception e) {
    System.out.println("Exception handled");
}

1️⃣5️⃣ Real-World Design – User Roles

Question:
Design a system where Admin, User, and Guest have different permissions. Which OOP concepts will you use?

Answer:
I will use inheritance for user roles, encapsulation for access control, and polymorphism for different behaviors.

class User {
    void access() {
        System.out.println("User access");
    }
}

class Admin extends User {
    void access() {
        System.out.println("Admin access");
    }
}

class Guest extends User {
    void access() {
        System.out.println("Guest access");
    }
}
