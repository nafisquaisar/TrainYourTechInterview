📘 Top 50 DBMS Interview Questions with Answers

1️⃣ Basics of DBMS

Q1. What is DBMS?
A Database Management System (DBMS) is software that manages and organizes data, providing storage, retrieval, security, and consistency.

Q2. Difference between DBMS and RDBMS?

  • DBMS → Data stored as files, no relations.
  • RDBMS → Data stored in tables with relationships (e.g., MySQL, Oracle).

Q3. What is a database schema?
A logical structure of the database defining tables, fields, relationships, and constraints.

Q4. What are the advantages of DBMS?

  • Reduced redundancy
  • Data integrity
  • Security
  • Concurrent access

Q5. What is data abstraction?
Hiding implementation details from users (Levels: Internal, Conceptual, External).

2️⃣ Keys & Constraints

Q6. What is a primary key?
A unique identifier for each record in a table.

Q7. Difference between candidate key and super key?

  • Candidate Key → Minimal set of attributes that can be primary key.
  • Super Key → Any set of attributes that uniquely identifies a record.

Q8. What is a foreign key?
An attribute that refers to the primary key of another table, establishing a relationship.

Q9. What are integrity constraints?

  • Domain Constraint
  • Key Constraint
  • Entity Integrity (PK not null)
  • Referential Integrity (valid FK)

Q10. What is a composite key?
A primary key formed by combining two or more attributes.

3️⃣ SQL Queries

Q11. Difference between DDL, DML, DCL, TCL?

  • DDL → CREATE, ALTER, DROP
  • DML → INSERT, UPDATE, DELETE
  • DCL → GRANT, REVOKE
  • TCL → COMMIT, ROLLBACK, SAVEPOINT

Q12. Write a query to fetch second highest salary.

SELECT MAX(salary) 
FROM employees 
WHERE salary < (SELECT MAX(salary) FROM employees);

Q13. What is the difference between WHERE and HAVING?

  • WHERE → Filters rows before grouping.
  • HAVING → Filters groups after aggregation.

Q14. What is a join?
A join combines rows from two or more tables based on related columns.

Q15. Types of joins in SQL?

  • INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN, CROSS JOIN, SELF JOIN.

Q16. What is a view?
A virtual table created using SQL queries.

Q17. What is an index?
A data structure that improves retrieval speed.

Q18. Difference between TRUNCATE, DELETE, and DROP?

  • DELETE → Removes rows, can use WHERE.
  • TRUNCATE → Removes all rows, cannot rollback.
  • DROP → Deletes entire table structure.

Q19. What is normalization?
Process of organizing data to reduce redundancy and improve integrity.

Q20. Write a query to find employees with salary greater than average.

SELECT * FROM employees 
WHERE salary > (SELECT AVG(salary) FROM employees);

4️⃣ Normalization

Q21. Explain 1NF.
Values must be atomic (no repeating groups).

Q22. Explain 2NF.
1NF + no partial dependency.

Q23. Explain 3NF.
2NF + no transitive dependency.

Q24. What is BCNF?
Every determinant must be a candidate key.

Q25. What is denormalization?
Introducing redundancy to improve query performance.

5️⃣ Transactions & Concurrency

Q26. What is a transaction?
A sequence of operations performed as a single logical unit of work.

Q27. What are ACID properties?

  • Atomicity, Consistency, Isolation, Durability.

Q28. What is a deadlock?
A situation where two or more transactions wait indefinitely for resources held by each other.

Q29. What is serializability?
Ensures concurrent transactions produce the same result as if executed serially.

Q30. What are concurrency issues?

  • Lost Update
  • Dirty Read
  • Non-repeatable Read
  • Phantom Read

Q31. What is a lock? Types?

  • Shared Lock → Multiple reads allowed.
  • Exclusive Lock → Only one write allowed.

Q32. What is checkpointing?
A mechanism to save DB state to minimize recovery time.

Q33. Difference between commit and rollback?

  • COMMIT → Save changes permanently.
  • ROLLBACK → Undo changes.

Q34. Difference between optimistic and pessimistic concurrency control?

  • Optimistic → Assumes no conflict, checks at commit.
  • Pessimistic → Locks resources to prevent conflict.

Q35. What is shadow paging?
A recovery technique using shadow copies of database pages.

6️⃣ Indexing & Storage

Q36. Difference between clustered and non-clustered index?

  • Clustered → Sorts and stores data physically.
  • Non-clustered → Logical ordering, separate structure.

Q37. Difference between dense and sparse index?

  • Dense → Every record has index entry.
  • Sparse → Only some records have entries.

Q38. What is a B+ Tree?
A balanced tree data structure used for indexing.

Q39. Difference between B Tree and B+ Tree?

  • B Tree → Keys + values in internal and leaf nodes.
  • B+ Tree → Only keys in internal nodes, values in leaf nodes.

Q40. What is RAID?
Redundant Array of Independent Disks → Improves reliability and performance.

7️⃣ Recovery & Advanced

Q41. What is log-based recovery?
Uses logs to redo or undo transactions after failure.

Q42. Difference between deferred and immediate update?

  • Deferred → Updates applied after commit.
  • Immediate → Updates applied instantly, logged for rollback.

Q43. What is distributed database?
Database distributed across multiple sites connected via network.

Q44. What is data replication?
Maintaining copies of data at multiple locations.

Q45. What is CAP theorem?
In distributed systems, only two of Consistency, Availability, and Partition tolerance can be guaranteed.

Q46. Difference between SQL and NoSQL databases?

  • SQL → Relational, structured, ACID.
  • NoSQL → Non-relational, flexible schema, scalable.

Q47. What is sharding?
Partitioning large database into smaller, faster, more manageable pieces.

Q48. Difference between OLTP and OLAP?

  • OLTP → Online Transaction Processing (real-time operations).
  • OLAP → Online Analytical Processing (analysis & reporting).

Q49. What is Big Data in context of DBMS?
Large datasets with Volume, Velocity, Variety → handled by Hadoop, Spark.

Q50. What are some common DBMS software?

  • MySQL, PostgreSQL, Oracle, SQL Server, MongoDB.
