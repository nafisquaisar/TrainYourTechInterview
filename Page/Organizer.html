1️⃣ Project Overview & Understanding

Q1. Can you explain your Organizer Classes app in simple words?

Answer:
Organizer Classes is an Android ed-tech application that provides study notes, video lectures, previous-year questions, mock tests, a doubt-solving section, and helps students check their progress.

The idea came from a real problem I personally observed.
While teaching students, especially Bihar Board English-medium students, I noticed that many of them struggled because they did not get proper previous-year questions and MCQs that are actually asked in exams.

To solve this problem, I built an app that organizes learning materials according to the board pattern.
This helps students prepare in a more focused and exam-oriented way.

Initially, I focused on Bihar Board students, but the app is also useful for CBSE and other state boards.

Q2. What problem does this app solve?

Answer:
Organizer Classes solves the problem of lack of exam-oriented study material, especially for Bihar Board English-medium students.

Many students do not get proper previous-year questions and MCQs, even though a large part of the exam is MCQ-based.
They depend on scattered resources, which creates confusion and weak exam preparation.

This app brings board-pattern notes, MCQs, PYQs, mock tests, and performance tracking into one platform, helping students prepare with clarity and confidence.

3️⃣ Firebase – Core Questions

🔐 Firebase Authentication

Q18. What is Firebase Authentication?

Answer:

  • Firebase Authentication is a service by Firebase that provides secure and easy user sign-in for apps without building a full authentication system from scratch.

  • It handles user identity, login sessions, and token management, while the developer focuses on app features.

  • It supports multiple sign-in methods like Email & Password, Phone OTP, Google, and Anonymous login, and provides a unique user ID for each user.

  • Firebase Authentication is secure, scalable, and integrates easily with Android apps.

Q19. Which login methods did you use?

Answer:

  • I used Email and Password authentication and Sign-in with Google for user login.

Q20. How does Firebase Authentication work internally?

Answer:

  1. Firebase Authentication works using a token-based authentication system.

  2. When a user signs in using Email, Google, or Phone,
    Firebase verifies the credentials with the selected identity provider.

  3. After successful verification, Firebase creates a user account and generates a unique UID for that user.

  4. Firebase then issues a secure ID token (JWT), which is stored on the device and automatically refreshed.

  5. This token is sent with each request to Firebase services like Firestore or Realtime Database to verify the user’s identity.

  6. Firebase Security Rules use the UID and token claims to allow or deny access to data.

UID = User ID (User Identifier)

JWT = JSON Web Token
  1. It is a secure token format used for authentication
  2. It contains user identity information in encrypted form
  3. Firebase uses JWT as an ID token to verify the user on each request

Q21. What is Firebase UID?

Answer:

  • Firebase UID is a unique ID given to every user by Firebase when they sign up.
  • It never changes and is used to identify the user and store their data across Firebase services.

Q22. How do you maintain user session?

Answer:

  • Firebase automatically maintains the session until logout.

🗄️ Firestore Database

Q25. What is Cloud Firestore?

Answer:
Cloud Firestore is a NoSQL database by Firebase that stores data in collections and documents and syncs data in real time.

Q26. Why did you choose Firestore over Realtime Database?

Answer:

  • I chose Cloud Firestore because it provides better data structure, scalability, and querying compared to Realtime Database.

  • Firestore stores data in collections and documents, which fits well with complex app data. It also supports powerful queries, indexing, and better security rules.

Q27. How is data stored in Firestore?

Answer:
Firestore stores data in collections, which contain documents, and each document holds key–value fields.

Q28. What are collections and documents?

Answer:
A collection is like a folder, and a document is like a file inside that folder.
Each document contains fields such as name, email, or age.

Q29. How did you structure Firestore data?

I structured Firestore data using separate collections such as classes, subjects, videos, and notes, with each document representing a single item.

Q30. How did you fetch data efficiently?

Answer:
I avoided loading all data at once and fetched only required data using queries and pagination.

🔄 Realtime Database

Q35. Why did you use Realtime Database?

Answer:
Realtime Database is ideal for fast, real-time synchronization, which is why I used it.

Q36. What type of data did you store in it?

Answer:
I stored real-time and frequently changing data in Firebase Realtime Database, such as class information, course tests, student doubts, and user-related data that needed instant updates.

Q37. Difference between Firestore and Realtime Database

Feature Cloud Firestore Realtime Database
Data model Collections and documents Single JSON tree
Queries Powerful queries (where, orderBy, limit) Limited filtering options
Scaling Better scalability for large apps Suitable for small to medium data
Indexing Automatic and composite indexes Manual and limited indexing
Offline support Strong built-in offline support Basic offline support
Use case Structured and complex data Instant real-time updates

Summary:

Cloud Firestore is better for structured, scalable, and complex data with advanced querying, while Realtime Database is mainly used for fast real-time synchronization with simpler data.

Q38. When should we prefer Realtime Database?

Answer:
Realtime Database is used when data must update instantly.

Q39. How does real-time update work?

Answer:
Realtime Database uses listeners to detect data changes.
Whenever data is added, updated, or deleted, the changes are immediately pushed to all connected clients without refreshing the app.
