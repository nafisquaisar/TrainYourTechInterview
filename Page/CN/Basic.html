Networking Basics, OSI & TCP/IP Models

🔹 Introduction to Computer Networking

  • Computer Network: A computer network is a group of computers and other devices connected together so they can share data, resources, and information.

Types of Computer Networks

1. PAN (Personal Area Network)

  • Range: Up to 10 meters
  • Description: Used to connect personal devices over a short distance
  • Examples: Bluetooth, mobile hotspot

2. LAN (Local Area Network)

  • Range: Up to 1 km
  • Description: Covers a small geographical area like a home, office, or school
  • Examples: Home Wi‑Fi, office network

3. WLAN (Wireless Local Area Network)

  • Range: 30–100 meters
  • Description: Wireless version of LAN
  • Examples: Wi‑Fi network

4. MAN (Metropolitan Area Network)

  • Range: 5–50 km
  • Description: Covers a city or metropolitan area
  • Examples: City cable network, metro Wi‑Fi

5. WAN (Wide Area Network)

  • Range: 100 km to worldwide
  • Description: Covers a large geographical area across countries or continents
  • Examples: Internet, bank networks

🔹 Network Topologies

The arrangement of nodes (computers/devices) in a network.

🔹 Bus Topology

  • Bus Topology: Single central cable; simple but failure affects entire network.

Real-life example: Electricity line in a street

  • One main cable acts like a single power line
  • Many houses/devices connect to it
  • If the main line fails, the whole area loses power

📌 Similar to old computer labs using one long cable

🔹 Star Topology

  • Star Topology: All nodes connected to a central hub/switch; easy to manage.

Real-life example: Home or office Wi-Fi router

  • Router acts as a central hub
  • All devices connect directly to the router
  • If one device fails, others still work
  • If the router fails, the whole network stops

📌 Most commonly used topology today

🔹 Mesh Topology

  • Mesh Topology: Every node connects to every other; highly reliable but costly.

Real-life example: Road network in a big city

  • Many alternate paths exist between locations
  • If one road is blocked, traffic uses another
  • Highly reliable but costly to maintain

📌 Used in military systems and internet backbone

🔹 Ring Topology

  • Ring Topology: Each device connects to two others in a circular path.

Real-life example: People sitting in a circle passing a message

  • Message travels in a circular direction
  • Each person forwards the message
  • If one person leaves, communication breaks

📌 Used in token-based networks

🔹 Hybrid Topology

  • Hybrid Topology: Combination of two or more topologies.

Real-life example: School or college campus network

  • Each classroom uses star topology
  • All classrooms connect to a main backbone
  • Combination of different topologies

📌 Most real-world networks are hybrid

🔹 Network Architecture

Network architecture means how computers are connected in a network and how they communicate with each other.

1. Client–Server Architecture

  • One computer works as a server
  • Other computers are clients
  • Clients request data, server provides data

Example: Websites, email, online banking

Simple idea: Many users depend on one main computer

2. Peer-to-Peer (P2P) Architecture

  • All computers are equal
  • No central server
  • Each computer can send and receive data

Example: File sharing, small home networks

Simple idea: All computers share data with each other

🔹 OSI Model (Open Systems Interconnection)

  • Definition: A 7-layer conceptual model that standardizes communication between systems.

The 7 Layers:

  1. Physical Layer (Data in binary data (0 and 1) ) → Transmits raw bits over physical medium (cables, signals).
  2. Data Link Layer (Data in frame) → Provides error detection, MAC addressing, frames (Ethernet, PPP).
  3. Network Layer (Data in Packet) → Provides logical addressing & routing (IP, ICMP, ARP).
  4. Transport Layer (Data in segment) → Reliable/unreliable delivery, segmentation (TCP, UDP).
  5. Session Layer → Establishes, maintains, ends communication sessions.
  6. Presentation Layer → Data translation, compression, encryption.
  7. Application Layer → User interface protocols (HTTP, FTP, DNS, SMTP).

🔹 TCP/IP Model

  • A practical model used in real-world networking.

  • Has 4 layers:

    1. Application Layer → Combines OSI’s Application + Presentation + Session (HTTP, FTP, DNS).
    2. Transport Layer → Same as OSI’s (TCP, UDP).
    3. Internet Layer → Same as OSI’s Network layer (IP, ICMP).
    4. Network Access Layer → Combination of OSI’s Physical + Data Link layers.

🔹 OSI Model vs TCP/IP Model

Feature OSI Model TCP/IP Model
Layers 7 4
Developed By ISO DARPA (U.S. Dept. of Defense)
Usage Conceptual/reference model Practical/real-world model
Protocols General standard Specific set of protocols (TCP, IP, etc.)

🔹 Encapsulation & Decapsulation

  • Encapsulation (Sending Data): Process of wrapping data with protocol information at each layer before transmission.

    • Application Data → Segment (Transport Layer) → Packet (Network Layer) → Frame (Data Link Layer) → Bits (Physical Layer)

  • Decapsulation (Receiving Data): Reverse process of removing headers/trailers at each layer to retrieve original data.
