🌐 IP Address Notes

1. What is an IP Address? 🖥️

  • IP Address stands for Internet Protocol Address.
  • Unique identifier for a device on a network (LAN or Internet).
  • Enables devices to communicate with each other over a network.

2. Types of IP Address 🏷️

a) Based on Version 🔢

  1. IPv4

    • 32-bit address (e.g., 192.168.1.1)
    • Format: A.B.C.D where each is 0–255
    • Total addresses: ~4.3 billion 🌍

  2. IPv6

    • 128-bit address (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334)
    • Solves IPv4 exhaustion ✅
    • Hexadecimal notation with colons

b) Based on Scope 🌐

  1. Private IP 🏠

    • Used within LAN

    • Not routable on the Internet ❌

    • Examples:

      • Class A: 10.0.0.0 – 10.255.255.255
      • Class B: 172.16.0.0 – 172.31.255.255
      • Class C: 192.168.0.0 – 192.168.255.255

  2. Public IP 🌎

    • Used on the Internet
    • Assigned by ISP
    • Must be unique globally ✅

      • Class A: 1.0.0.0 – 9.255.255.255 and 11.0.0.0 – 126.255.255.255 (except private/reserved ones)

      • Class B: 128.0.0.0 – 172.15.255.255 and 172.32.0.0 – 191.255.255.255

      • Class C: 192.0.0.0 – 192.167.255.255 and 192.169.0.0 – 223.255.255.255

c) Based on Assignment 🔧

  1. Static IP 📌

    • Fixed address
    • Example: Servers, printers

  2. Dynamic IP 🔄

    • Changes periodically
    • Assigned by DHCP

3. IP Address Classes (IPv4) 📊

Class Range Default Subnet Mask Usage
A 0–127 255.0.0.0 Large networks (few networks, many hosts)
B 128–191 255.255.0.0 Medium networks
C 192–223 255.255.255.0 Small networks
D 224–239 N/A Multicast 📡
E 240–255 N/A Experimental / research 🔬

Class A

  • Range: 1–126 (0 and 127 are reserved)

    • 0.x.x.x → Reserved for default route / special use
    • 127.x.x.x → Loopback addresses (used for testing on local machine)

  • Default Subnet Mask: 255.0.0.0

  • Usage: Very large networks, few networks but many hosts per network

  • Hosts per network: ~16 million

  • Network ID / Host ID in binary: First 8 bits network, remaining 24 bits host ID

    • Example: 00000001.00000000.00000000.00000000

Class B

  • Range: 128–191

  • Default Subnet Mask: 255.255.0.0

  • Usage: Medium-sized networks

  • Hosts per network: ~65,000

  • Network ID / Host ID in binary: First 16 bits network, remaining 16 bits host ID

    • Example: 10000000.00000001.00000000.00000000

Class C

  • Range: 192–223

  • Default Subnet Mask: 255.255.255.0

  • Usage: Small networks

  • Hosts per network: 254

  • Network ID / Host ID in binary: First 24 bits network, remaining 8 bits host ID

    • Example: 11000000.10101000.00000001.00000000

Class D

  • Range: 224–239
  • Usage: Multicast addresses 📡
  • Hosts: Not used for standard hosts
  • Binary Representation: Not applicable for network/host division

Class E

  • Range: 240–255
  • Usage: Experimental / research 🔬
  • Hosts: Not used for standard hosts
  • Binary Representation: Not applicable for network/host division

🔹 APIPA (Automatic Private IP Addressing)

  • Range: 169.254.0.0/16 → (169.254.0.0 – 169.254.255.255)
  • When used: A host assigns an APIPA address to itself when it cannot get an IP from DHCP.
  • Routability: ❌ Not routable — stays on the local link only.
  • Example: Host picks 169.254.10.5/16 → Network 169.254.0.0, Broadcast 169.254.255.255.

🔁 Loopback (Local Testing)

  • Range: 127.0.0.0/8 (commonly 127.0.0.1)
  • Purpose: Test the TCP/IP stack of the local machine.
  • Notes: Traffic never leaves the host. 127.x.x.x always returns to the same machine.

🏁 Network Address (all host bits = 0)

  • Definition: For a given IP + subnet mask, the network address is the IP with all host bits set to 0.
  • Assignable? ❌ Not assignable to hosts, identifies the network itself.
  • Example: 192.168.1.101/24 → Network = 192.168.1.0.

📣 Broadcast Address (all host bits = 1)

  • Definition: Sets all host bits to 1, used to send data to all hosts on a network.

  • Types:

    • Limited Broadcast: 255.255.255.255 (local network only, never routed).
    • Directed Broadcast: For 192.168.1.0/24 → Broadcast = 192.168.1.255.

  • Routers: Often block directed broadcasts for security.

✅ How to Find Network & Broadcast

  1. Write IP & Mask in binary.
  2. Network = IP AND Mask.
  3. Broadcast = Network OR (inverse Mask).
  4. Convert back to decimal.

Example: IP 192.168.1.101/24 → Network = 192.168.1.0, Broadcast = 192.168.1.255.

🎭 Subnet Mask

  • Definition: 32-bit number separating network part & host part of an IP.

  • Format: Dotted decimal (255.255.255.0) or CIDR (/24).

  • Example:

    • IP: 192.168.1.10
    • Mask: 255.255.255.0 → /24
    • Network: 192.168.1.0
    • Host Range: 192.168.1.1 – 192.168.1.254

👉 Subnet mask = defines network size.

✂️ Subnetting

  • Definition: Dividing a large network into smaller subnetworks.

  • Benefits: Efficient IP usage, security, reduced broadcast traffic.

  • Formulas:

    • Subnets = 2^borrowed_bits
    • Hosts per subnet = 2^host_bits - 2

  • Example:

    • Network: 192.168.1.0/24 → 256 IPs.

    • Subnet into /26 → 4 subnets (64 IPs each):

      • 192.168.1.0 – 192.168.1.63
      • 192.168.1.64 – 192.168.1.127
      • 192.168.1.128 – 192.168.1.191
      • 192.168.1.192 – 192.168.1.255

👉 Subnetting = cutting networks into smaller pieces.

🔄 NAT (Network Address Translation)

  • Purpose: Translates private ↔ public IPs.

  • Why: Private IPs are not routable, NAT enables Internet access.

  • Types:

    • Static NAT: One private ↔ One public (1:1).
    • Dynamic NAT: Private ↔ Public chosen from a pool.
    • PAT (NAT Overload): Many private IPs share one public IP (differentiated by ports).

  • Example: 192.168.1.2 → NAT → Public 103.25.33.5 → Internet.

👉 NAT = bridge between private networks and the Internet.

🚀 Classless IP Address (CIDR)

  • Definition: Classless Inter-Domain Routing → removes old Class A/B/C system.

  • Format: IP/prefix_length.

  • Why: Saves IPs, avoids wastage, improves routing.

  • Example:

    • Old Class C: 192.168.1.0 → Mask 255.255.255.0 (/24) → 256 IPs.
    • With CIDR: 192.168.1.0/26 → 64 IPs.

  • Benefits:

    • Efficient address allocation ✅
    • Smaller routing tables ✅

👉 CIDR = flexible addressing with /x prefix.

📝 Quick Memory Hook

  • APIPA: 169.254.0.0/16 → DHCP failsafe.
  • Loopback: 127.0.0.0/8 → local host testing.
  • Network: Host bits = 0 → identifies network.
  • Broadcast: Host bits = 1 → reach all hosts.
  • Subnet Mask: Divides network/host.
  • Subnetting: Splits networks.
  • NAT: Private ↔ Public IP conversion.
  • CIDR: Classless, efficient, /x.

5. Special IP Addresses ⚠️

Type Address Use Case
Loopback 🔁 127.0.0.1 Test network on local machine
Default Gateway 🚪 Usually first IP in subnet Device to reach other networks
Broadcast 📣 Last IP in subnet Send data to all devices in subnet
APIPA 🤖 169.254.x.x Automatic when DHCP fails

6. Private vs Public IP Example 🏢

Scenario: Office network

  • Private IP: 192.168.1.10 (computer inside LAN) 💻
  • Public IP: 103.45.21.10 (router's Internet IP) 🌐
  • Computer can access the Internet via NAT (Network Address Translation) 🔄

7. Dynamic IP Scenario 🔄

  • User’s laptop connects to home Wi-Fi 📶
  • Router assigns 192.168.1.15 today
  • Tomorrow, DHCP assigns 192.168.1.20
  • Internet sees the router’s public IP 🌍

8. Static IP Scenario 📌

  • Company server: 192.168.1.50
  • Needed so that other devices can always reach it 🔗
  • Configured manually ✍️

9. IP Address Troubleshooting 🛠️

  1. Ping: Check if a host is reachable (ping 8.8.8.8) 🏓
  2. ipconfig / ifconfig: View current IP, subnet, gateway 💻
  3. Traceroute: Shows path packets take 🛣️
  4. Conflict Scenario: Two devices have same static IP → network error ⚠️

10. Real-World Scenarios 🌟

  1. Office LAN 🏢

    • Router: 192.168.1.1
    • PCs: 192.168.1.2 – 192.168.1.50
    • Printer: 192.168.1.100
    • Network uses DHCP 🔄

  2. Home Network 🏠

    • Router public IP from ISP 🌐
    • Devices inside: private IPs 💻📱
    • NAT allows Internet access 🔄

  3. Server Hosting 🌐

    • Web server requires static public IP 📌
    • Ensures clients can always access ✅

11. Key Points 📌

  • IPv4 still most widely used, IPv6 growing 🚀
  • Private IPs + NAT = Internet access for LAN 🔄
  • Always check subnet mask for correct networking ✅
  • Dynamic IP is convenient, static IP needed for servers 📌
  • Special addresses (loopback, broadcast) are essential for networking ⚠️
