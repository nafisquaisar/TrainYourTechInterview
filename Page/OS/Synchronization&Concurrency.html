🔐 Synchronization & Concurrency – Complete Notes (GATE ⭐⭐⭐ & Interview Oriented)

1. What is Synchronization?

Synchronization is the coordination of concurrent processes or threads to ensure correct and consistent access to shared resources.

📌 One-line (GATE / Interview):

Synchronization prevents race conditions while accessing shared data.

2. Concurrency

Concurrency means multiple processes or threads making progress simultaneously.

  • Can be achieved using time-sharing (single CPU)
  • Or parallelism (multiple CPUs)

📌 GATE Note: Concurrency ≠ Parallelism (conceptual difference)

3. Critical Section Problem ⭐⭐⭐

A critical section is a part of program where shared resources (variables, files, memory) are accessed.

Requirements for Solution

  1. 🔐 Mutual Exclusion – Only one process enters CS at a time
  2. ⚙️ Progress – If no one is in CS, one should be allowed to enter
  3. ⏳ Bounded Waiting – No process should wait indefinitely

📌 GATE Favorite: All 3 conditions must be satisfied

4. Race Condition

A race condition occurs when outcome depends on the order of execution of processes.

➡ Happens if critical section is not protected.

5. Mutex vs Semaphore ⭐⭐⭐

Mutex Semaphore
Locking mechanism Signaling mechanism
Binary (0/1) Binary or counting
Owned by thread No ownership
Used by same thread Any process can signal

📌 Interview Trap: Mutex has ownership, semaphore doesn’t

6. Binary vs Counting Semaphore ⭐⭐⭐

Binary Semaphore

  • Values: 0 or 1
  • Similar to mutex
  • Used for mutual exclusion

Counting Semaphore

  • Values: 0 to n
  • Used when multiple resources exist

7. Semaphore Value Meaning ⭐⭐⭐ (Very Important)

Let semaphore value = S

  • 🟢 S > 0 → S resources available
  • 🟡 S = 0 → No resource free, process must wait
  • 🔴 S < 0 → |S| processes are blocked

📌 GATE Favorite Question Area

8. Semaphore Operations

  • wait(P) → decrement semaphore
  • signal(V) → increment semaphore
wait(S):
  S = S − 1
  if S < 0 → block

signal(S):
  S = S + 1
  if S ≤ 0 → wake up a process

9. Busy Waiting vs Blocking ⭐⭐⭐

Busy Waiting Blocking
Process keeps spinning Process sleeps
Wastes CPU cycles CPU efficient
Used in spinlocks Used in semaphores

📌 Interview Line: Busy waiting wastes CPU

10. Producer–Consumer Problem ⭐⭐⭐

Classic synchronization problem with shared buffer.

Tools Used

  • 🔐 Mutex → protect buffer
  • 📦 Empty → count empty slots
  • 📦 Full → count filled slots

📌 GATE Note: Requires counting semaphore

11. Dining Philosopher Problem ⭐⭐⭐

A classic problem illustrating deadlock and starvation.

Problem

  • 5 philosophers
  • 5 forks
  • Each needs 2 forks to eat

Issues

  • Deadlock possible
  • Starvation possible

Solutions

  • Limit number of philosophers
  • Asymmetric pickup
  • Use semaphore

📌 Interview Favorite Problem

12. Monitor (High-level Synchronization)

A monitor is a high-level synchronization construct that:

  • Provides mutual exclusion automatically
  • Uses condition variables

📌 GATE Note: Easier than semaphore
