🔗 Linked List – Complete Notes

📌 What is a Linked List?

A Linked List is a linear data structure where elements are stored in nodes.
Each node contains:

  • Data
  • Reference (pointer) to the next node

Unlike arrays, linked lists do not store data in contiguous memory.

📦 Structure of a Node 

[ Data | Next ]
  • Data → actual value
  • Next → address of the next node
  • First node is called Head
  • Last node points to NULL

🔹 Types of Linked Lists

1️⃣ Singly Linked List

  • Each node points to the next node only
  • One-direction traversal
Head → A → B → C → NULL

Use when: memory is limited and traversal is simple.

2️⃣ Doubly Linked List

  • Each node has two pointers:

    • Previous
    • Next
NULL ← A ⇄ B ⇄ C → NULL

Use when: backward traversal is required.

3️⃣ Circular Linked List

  • Last node points back to the first node
A → B → C
↑         ↓
← ← ← ← ←

Use when: continuous looping is required (e.g. playlists, scheduling).

⚙️ Common Operations

🔸 Traversal

  • Visit each node one by one
  • Time Complexity: O(n)

🔸 Insertion

Type Time
At beginning O(1)
At end O(n)
At position O(n)

🔸 Deletion

Type Time
At beginning O(1)
At end O(n)
Specific node O(n)

🔸 Searching

  • Time Complexity: O(n)

🧮 Time Complexity Summary

Operation Time
Access O(n)
Insert (start) O(1)
Insert (end) O(n)
Delete O(n)
Search O(n)

📊 Linked List vs Array

Feature Array Linked List
Memory Contiguous Non-contiguous
Size Fixed Dynamic
Insertion Costly Easy
Deletion Costly Easy
Access O(1) O(n)

✅ Advantages

  • Dynamic size
  • Efficient insertion and deletion
  • Better memory utilization

❌ Disadvantages

  • Extra memory needed for pointers
  • No random access
  • Slower traversal compared to arrays

🎯 Real-World Applications

  • Music playlist
  • Browser back and forward navigation
  • Undo and redo operations
  • Implementing stacks and queues

📝 Interview Tip

Why Linked List over Array?

Linked lists support dynamic memory allocation and allow efficient insertion or deletion without shifting elements.
