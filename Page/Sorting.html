✅ 1. Selection Sort 🔍

  • Selection Sort is a simple sorting algorithm where we select the smallest element from the unsorted part of the array and place it at the correct position.
In simple words:
  • Selection Sort repeatedly finds the minimum element and puts it at the beginning.

Intuition 🤔

  • At each step, find the minimum 🔽 element from the unsorted part 🔄 and place it at the beginning 🟩.
  • Unsorted area shrinks by 1️⃣ each time.

Example 🧪

Input: {13, 46, 24, 52, 20, 9}
Output: 9, 13, 20, 24, 46, 52

Code 💻

for (int i = 0; i < n - 1; i++) {
    int mini = i;
    for (int j = i + 1; j < n; j++) {
        if (arr[j] < arr[mini]) mini = j;
    }
    int temp = arr[mini];
    arr[mini] = arr[i];
    arr[i] = temp;
}

⏱️ Time: O(n²)

📌 Stable: ❌

📦 In-place: ✔️

🌐 2. Bubble Sort 🫧

  • Bubble Sort is a simple sorting algorithm where we repeatedly compare adjacent elements and swap them if they are in the wrong order. Larger elements gradually move to the end, like bubbles rising to the top.

Intuition 🤔

  • Swap adjacent elements 🔁 if they are in the wrong order 🚫.
  • The largest element bubbles up ⬆️ each pass.

Code 💻

for (int i = 0; i < n - 1; i++) {
    for (int j = 0; j < n - i - 1; j++) {
        if (arr[j] > arr[j + 1]) {
            int temp = arr[j];
            arr[j] = arr[j + 1];
            arr[j + 1] = temp;
        }
    }
}

⏱️ Time: O(n²)

📌 Stable: ✔️

📦 In-place: ✔️

🎓 3. Insertion Sort ✍️

  • Insertion Sort is a simple sorting algorithm where we insert each element into its correct position in the already sorted part of the array.

Intuition 🤔

  • Insert each element in its correct sorted position 📥.
  • Similar to sorting playing cards 🃏.

Code 💻

for (int i = 1; i < n; i++) {
    int temp = arr[i], j = i - 1;
    while (j >= 0 && arr[j] > temp) {
        arr[j + 1] = arr[j];
        j--;
    }
    arr[j + 1] = temp;
}

⏱️ Time: O(n²)

📌 Stable: ✔️

📦 In-place: ✔️

🔄 4. Merge Sort 🧩

  • Merge Sort is a sorting algorithm that uses the divide and conquer approach.
In simple words:
  • It divides the array into smaller parts, sorts them, and then merges them back in order.

Intuition 🤔

  • Divide ➗, Sort 📊, Merge 🔗.
  • Uses recursion 🔁.

Code 💻

void mergeSort(int[] arr, int low, int high) {
    if (low >= high) return;
    int mid = (low + high) / 2;
    mergeSort(arr, low, mid);
    mergeSort(arr, mid + 1, high);
    merge(arr, low, mid, high);
}

Merge Logic 🔗

void merge(int[] arr, int low, int mid, int high) {
    List<Integer> temp = new ArrayList<>();
    int left = low, right = mid + 1;
    while (left <= mid && right <= high) {
        temp.add(arr[left] <= arr[right] ? arr[left++] : arr[right++]);
    }
    while (left <= mid) temp.add(arr[left++]);
    while (right <= high) temp.add(arr[right++]);
    for (int i = low; i <= high; i++) arr[i] = temp.get(i - low);
}

⏱️ Time: O(n log n)

📌 Stable: ✔️

📦 In-place: ❌

🔪 5. Quick Sort ⚡

Intuition 🤔

  • Pick a pivot 🎯.
  • Elements smaller ⬅️ go left, greater ➡️ go right.
  • Recursively sort.

Code 💻

void quickSort(List<Integer> arr, int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}

Partition Logic 🌗

int partition(List<Integer> arr, int low, int high) {
    int pivot = arr.get(low);
    int i = low, j = high;
    while (i < j) {
        while (i <= high - 1 && arr.get(i) <= pivot) i++;
        while (j >= low + 1 && arr.get(j) > pivot) j--;
        if (i < j) Collections.swap(arr, i, j);
    }
    Collections.swap(arr, low, j);
    return j;
}

⏱️ Time:

  • Best/Average 🟢: O(n log n)
  • Worst 🔴: O(n²)

📌 Stable: ❌

📦 In-place: ✔️

🔼 6. Heap Sort ⛰️

Intuition 🤔

  • Build a max heap 🏔️.
  • Swap root with last element 🔁.
  • Heapify 🔄.

Code 💻

static void heapSort(int[] arr) {
        int n = arr.length;

        // Build heap (rearrange vector)
        for (int i = n / 2 - 1; i >= 0; i--)
            heapify(arr, n, i);

        // One by one extract an element from heap
        for (int i = n - 1; i > 0; i--) {

            // Move current root to end
            int temp = arr[0];
            arr[0] = arr[i];
            arr[i] = temp;

            // Call max heapify on the reduced heap
            heapify(arr, i, 0);
        }
    }


static void heapify(int[] arr, int n, int i) {
    int largest = i;
    int l = 2 * i + 1;
    int r = 2 * i + 2;

    if (l < n && arr[l] > arr[largest]) largest = l;
    if (r < n && arr[r] > arr[largest]) largest = r;

    if (largest != i) {
        int temp = arr[i]; arr[i] = arr[largest]; arr[largest] = temp;
        heapify(arr, n, largest);
    }
}

⏱️ Time: O(n log n)

📌 Stable: ❌

📦 In-place: ✔️

🔢 7. Counting Sort 🔟

Intuition 🤔

  • Count occurrences 🔢.
  • Accumulate ➕.
  • Place in output array 📤.

Full Code 💻 (Java)

static int[] countingSort(int[] arr) {
    int n = arr.length;

    // Find max value
    int max = arr[0];
    for (int i = 1; i < n; i++) {
        if (arr[i] > max) max = arr[i];
    }

    // Create count array
    int[] count = new int[max + 1];
    for (int num : arr) {
        count[num]++; // Count occurrences
    }

    // Prefix sum (accumulate)
    for (int i = 1; i <= max; i++) {
        count[i] += count[i - 1];
    }

    // Output array
    int[] output = new int[n];
    for (int i = n - 1; i >= 0; i--) {
        output[count[arr[i]] - 1] = arr[i];
        count[arr[i]]--;
    }

    return output;
}

⏱️ Time: O(n + k)

📌 Stable: ✔️

📦 In-place: ❌

🔁 8. Radix Sort 🎚️

Intuition 🤔

  • Sort digits from least → most significant 🔢.
  • Uses stable counting sort internally.

Full Code 💻 (Java)

// Radix Sort function
static void radixSort(int[] arr) {
    int n = arr.length;

    // Find max to know number of digits
    int max = arr[0];
    for (int i = 1; i < n; i++) {
        if (arr[i] > max) max = arr[i];
    }

    // Apply counting sort for every digit
    for (int exp = 1; max / exp > 0; exp *= 10) {
        countingSortByDigit(arr, exp);
    }
}

// Counting sort for a specific digit
static void countingSortByDigit(int[] arr, int exp) {
    int n = arr.length;
    int[] output = new int[n];
    int[] count = new int[10]; // digits 0-9

    // Count occurrences of digits
    for (int i = 0; i < n; i++) {
        int digit = (arr[i] / exp) % 10;
        count[digit]++;
    }

    // Prefix sum
    for (int i = 1; i < 10; i++) {
        count[i] += count[i - 1];
    }

    // Build output (iterate backward to maintain stability)
    for (int i = n - 1; i >= 0; i--) {
        int digit = (arr[i] / exp) % 10;
        output[count[digit] - 1] = arr[i];
        count[digit]--;
    }

    // Copy back
    for (int i = 0; i < n; i++) arr[i] = output[i];
}

⏱️ Time: O(d × (n + b))

📌 Stable: ✔️

📦 In-place: ❌

📌 Key Takeaways

  • Quick Sort is fastest in most practical cases
  • Merge Sort gives guaranteed performance and is stable
  • Insertion Sort works best for nearly sorted arrays
  • Heap Sort is good when memory usage matters
  • Bubble Sort is mainly for learning

🎯 Interview One-Line Answer

I choose the sorting algorithm based on data size and requirement. Quick Sort for speed, Merge Sort for stability, and Insertion Sort for nearly sorted data.

✨ Tip for interviews 🎯:
Use Merge when stability matters, Quick for average speed, Heap for guaranteed time, and Counting/Radix for limited integer ranges.
