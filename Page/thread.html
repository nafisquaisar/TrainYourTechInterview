✅ Java Thread – Best Notes

📌 What is a Thread?

  • A thread is a lightweight sub-process, the smallest unit of processing.
  • Multithreading is a process of executing multiple threads simultaneously.
  • It uses a shared memory model and is more efficient than multiprocessing.

🧵 Ways to Create a Thread in Java

1. By Extending Thread class 

class MyThread extends Thread {
    public void run() {
        System.out.println("Thread is running...");
    }
}

public class Main {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        t1.start();  // start() calls run() in a new thread
    }
}

2. By Implementing Runnable interface 

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Thread is running...");
    }
}

public class Main {
    public static void main(String[] args) {
        Thread t1 = new Thread(new MyRunnable());
        t1.start();
    }
}

🔄 Thread Lifecycle

  1. New
  2. Runnable
  3. Running
  4. Blocked/Waiting
  5. Terminated/Dead

⚙️ Important Thread Methods

Method Description
start() Starts a thread
run() Entry point of thread logic
sleep(ms) Sleeps for ms milliseconds
join() Waits for thread to finish
yield() Pause and allow other threads
setPriority() Sets thread priority
isAlive() Checks if thread is active
interrupt() Interrupts a sleeping/waiting thread

The sleep() method in Java throws a checked exception:
🛑 InterruptedException

🧠 Thread Priority

  • Range: 1 (MIN_PRIORITY) to 10 (MAX_PRIORITY)
  • Default: 5 (NORM_PRIORITY)

🔒 Synchronization in Threads

Used to prevent race conditions when multiple threads access shared resources.

synchronized void sharedMethod() {
    // critical section
}

Or use synchronized block:

synchronized(this) {
    // critical section
}

⏳ Thread.sleep() vs wait()

Feature sleep() wait()
Class Thread Object
Lock Release ❌ Does not release lock ✅ Releases lock
Use Pause execution Inter-thread communication

🧪 Thread-safe Class Examples:

  • Vector, StringBuffer, Hashtable – synchronized
  • Use Collections.synchronizedList() for list thread-safety

🧪 Example: Thread with Shared Resource 

class Counter {
    int count = 0;
    synchronized void increment() {
        count++;
    }
}

public class Main {
    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();

        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) counter.increment();
        });

        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) counter.increment();
        });

        t1.start();
        t2.start();

        t1.join();
        t2.join();

        System.out.println("Count: " + counter.count);
    }
}

💬 Inter-Thread Communication

Methods: wait(), notify(), notifyAll()
Used for communication between threads.

synchronized(obj) {
    obj.wait();     // release lock and wait
    obj.notify();   // wake up waiting thread
}

✅ Best Practices

  • Prefer Runnable over extending Thread (for better flexibility).
  • Use ExecutorService for thread pool management.
  • Avoid using Thread.stop() – it's unsafe.
  • Always handle InterruptedException in sleep/wait.

👉 Java Daemon Thread

🔍 What is a Daemon Thread?

  • A Daemon thread is a background service thread that provides services to user threads.
  • It runs in the background and does not prevent the JVM from exiting.
  • When all user threads die, the JVM exits and kills any remaining daemon threads.

✨ Examples of Daemon Threads:

  • Garbage Collector
  • Finalizer Thread
  • Signal Dispatcher Thread

🔄 Daemon vs User Thread

Feature User Thread Daemon Thread
Runs in background? ❌ No ✅ Yes
Keeps JVM alive? ✅ Yes ❌ No
Purpose Main task execution Background services

⚖️ How to Make a Thread Daemon

  • Use setDaemon(true) before calling start().
class MyDaemon extends Thread {
    public void run() {
        while (true) {
            System.out.println("Daemon thread running...");
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

public class Main {
    public static void main(String[] args) {
        MyDaemon t = new MyDaemon();
        t.setDaemon(true);  // Must be called before start()
        t.start();

        System.out.println("Main thread ending...");
    }
}

❌ Important Points:

  • Must call setDaemon(true) before start() — otherwise it throws IllegalThreadStateException.
  • If all user threads finish, JVM exits even if daemon threads are running.
  • Daemon threads are suitable for background tasks like monitoring/logging/cleanup.

🔧 Checking if a Thread is Daemon: 

Thread t = new Thread();
t.setDaemon(true);
System.out.println(t.isDaemon());  // true
