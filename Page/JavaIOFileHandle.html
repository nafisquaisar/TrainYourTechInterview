📘 Java Input/Output (I/O) & File Handling – Clean Notes

A simple, structured, and interview-focused guide to Java I/O concepts including streams, console input/output, file handling, and best practices.

1️⃣ Introduction to Java I/O

  • I/O (Input/Output) is used to read data from a source and write data to a destination.
  • Java uses the concept of streams to handle I/O.
  • A stream is a flow of data.

🔹 Types of Streams

  • Byte Streams → handle binary data
  • Character Streams → handle text data

2️⃣ Byte Streams

  • Used for binary data like images, audio, video, pdf files.
  • Based on InputStream and OutputStream abstract classes.

Common Byte Stream Classes

  • FileInputStream
  • FileOutputStream
  • BufferedInputStream
  • BufferedOutputStream
  • DataInputStream / DataOutputStream
  • ObjectInputStream / ObjectOutputStream

Example: Byte File Read 

FileInputStream fis = new FileInputStream("file.txt");
int i;
while ((i = fis.read()) != -1) {
    System.out.print((char) i);
}
fis.close();

3️⃣ Character Streams

  • Used for text data.
  • Supports Unicode characters.
  • Based on Reader and Writer abstract classes.

Common Character Stream Classes

  • FileReader
  • FileWriter
  • BufferedReader
  • BufferedWriter
  • PrintWriter

Example: Character File Read 

BufferedReader br = new BufferedReader(new FileReader("file.txt"));
String line;
while ((line = br.readLine()) != null) {
    System.out.println(line);
}
br.close();

4️⃣ Byte Stream vs Character Stream

Feature Byte Stream Character Stream
Data Type Binary Text
Unit Byte Character
Encoding No Yes
Examples FileInputStream FileReader

5️⃣ Standard I/O Streams (System Class)

Stream Description
System.in Input stream (keyboard)
System.out Output stream (console)
System.err Error output stream

System.out vs System.err

  • System.out → normal output
  • System.err → error messages

6️⃣ Console Input in Java

Using BufferedReader (Preferred for Speed) 

BufferedReader br = new BufferedReader(
    new InputStreamReader(System.in)
);
String input = br.readLine();

Stream Wrapping Chain

  • System.in → byte stream
  • InputStreamReader → converts to character stream
  • BufferedReader → adds buffering

Feature Scanner BufferedReader
Ease of Use Easy Moderate
Speed Slower Faster

8️⃣ File Handling in Java

File Class

  • Represents file or directory metadata
  • Does not read/write data
File file = new File("test.txt");
System.out.println(file.exists());

9️⃣ Writing to a Text File 

BufferedWriter bw = new BufferedWriter(new FileWriter("data.txt"));
bw.write("Hello Java");
bw.newLine();
bw.write("File Handling");
bw.close();

🔟 Reading from a Text File 

BufferedReader br = new BufferedReader(new FileReader("data.txt"));
String line;
while ((line = br.readLine()) != null) {
    System.out.println(line);
}
br.close();

1️⃣1️⃣ Binary File Handling

  • Used for images, audio, video.
  • Use byte streams only.
FileInputStream fis = new FileInputStream("a.jpg");
FileOutputStream fos = new FileOutputStream("b.jpg");
int i;
while ((i = fis.read()) != -1) {
    fos.write(i);
}
fis.close();
fos.close();

1️⃣3️⃣ Best Practices

  • Always close streams
  • Use try-with-resources when possible
  • Use buffered streams for better performance
  • Use correct stream based on file type

📝 Interview Key Points

  • Streams provide uniform I/O abstraction
  • Byte streams → binary data
  • Character streams → text data
  • Buffered streams improve performance
  • File class handles metadata only

✅ Final Takeaway

Java I/O uses streams to handle data efficiently. Choosing the correct stream type and using buffering and proper resource management ensures high performance and safe file operations.
