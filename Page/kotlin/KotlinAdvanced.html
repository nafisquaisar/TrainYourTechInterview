🔰 Introduction

These concepts are very important for real-world Kotlin development, especially Android interviews. They help you write safer, cleaner, and more efficient code.

1️⃣3️⃣ Null Safety (Very Important)

Kotlin is designed to reduce NullPointerException (NPE) by handling null values at compile time.

🔹 Nullable Types

By default, variables cannot hold null.

var name: String = "Nafis"
// name = null ❌ Error

var city: String? = null  // ✅ Nullable

? means the variable can store null.

🔹 Safe Call Operator (?.)

Used to safely access a property or function.

val length = name?.length

If name is null, result is null instead of crashing.

🔹 Elvis Operator (?😃

Provides a default value if expression is null.

val len = name?.length ?: 0

🔹 let

Definition:

let is used to execute a block of code only when the object is not null. Inside the block, the object is accessed using the keyword it. It is commonly used for null checks and safe calls.

name?.let {
    println(it)
}

🔹 lateinit

Used for non-null variables initialized later.

  • Only for var
  • Mostly used in Android
lateinit var userName: String

userName = "Nafis"
println(userName)

⚠️ Accessing before initialization causes crash.

🔹 lazy

Initializes value only when needed.

val data: String by lazy {
    "Loaded Data"
}
  • Used with val
  • Thread-safe by default

🔹 Platform Types

Used when Kotlin interacts with Java code.

String!  // Can be nullable or non-null

Kotlin trusts Java, so developer must handle null carefully.

1️⃣4️⃣ Lambda Expressions

A lambda is an anonymous function.

🔹 Lambda Syntax 

val sum = { a: Int, b: Int -> a + b }
println(sum(2, 3))

🔹 it Keyword

When lambda has only one parameter.

val nums = listOf(1, 2, 3)
nums.forEach { println(it) }

🔹 Lambda with Receiver

Allows direct access to receiver object.

val result = StringBuilder().apply {
    append("Hello")
    append(" Kotlin")
}

🔹 Anonymous Functions

Similar to lambda but with explicit return.

val square = fun(x: Int): Int {
    return x * x
}

1️⃣5️⃣ Higher-Order Functions

A function that:

  • Takes another function as parameter OR
  • Returns a function

🔹 Functions as Parameters 

fun calculate(a: Int, b: Int, op: (Int, Int) -> Int): Int {
    return op(a, b)
}

val result = calculate(2, 3) { x, y -> x + y }

🔹 Returning Functions 

fun operation(): (Int, Int) -> Int {
    return { a, b -> a * b }
}

🔹 Inline Higher-Order Functions

Reduces overhead by copying code at compile time.

inline fun greet(action: () -> Unit) {
    action()
}

🔹 noinline

Prevents lambda from being inlined.

inline fun test(noinline block: () -> Unit) {
    block()
}

🔹 crossinline

Prevents non-local returns from lambda.

inline fun demo(crossinline action: () -> Unit) {
    Runnable { action() }
}

1️⃣6️⃣ Scope Functions

Scope functions execute a block of code within the context of an object.

🔹 let

Definition:

let executes a block of code only when the object is not null.
The object is accessed using it, and the function returns the result of the lambda.
Commonly used for null checks and safe calls.

name?.let {
    println(it)
}

🔹 apply

Definition:

apply is used to configure or initialize an object. Inside the block, the object is referred to as this, and the function returns the same object after applying changes.

val user = User().apply {
    name = "Nafis"
}

🔹 also

Definition:

also is used when you want to perform some additional operations like logging or debugging without modifying the object. The object is accessed using it, and the original object is returned.

list.also {
    println(it.size)
}

🔹 run

Definition:

run executes a block of code on an object and returns the result of the last expression. Inside the block, the object is accessed using this. It is useful for calculations on an object.

val len = name?.run {
    length
}

🔹 with

Definition:

with is used when you want to operate on an object without calling methods on it repeatedly. Inside the block, the object is accessed using this, and it returns the lambda result.

with(user) {
    println(name)
}
