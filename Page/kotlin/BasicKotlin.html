🟢 KOTLIN BASICS (Easy Notes)

These notes are written in simple language, with real examples, so you can easily understand and revise.

1️⃣ Introduction to Kotlin

What is Kotlin?

  • Kotlin is a programming language used to write software
  • Mostly used for Android app development
  • Created by JetBrains
  • It works with Java and runs on JVM

👉 Think of Kotlin as a better and simpler version of Java.

Features of Kotlin

  • Less code than Java (clean & short)
  • Null safety (helps avoid app crashes)
  • Works with Java code easily
  • Supports OOP + Functional programming
  • Smart compiler (auto type casting)

Kotlin vs Java (Easy & Clear Notes)

Kotlin Java
Less code – same kaam kam lines me More code – zyada boilerplate
Built-in null safety (?, safe calls) NullPointerException runtime me
Modern language Older, traditional style
Type inference supported Explicit data types needed
Concise and readable syntax Verbose syntax
Fully interoperable with Java Works with Kotlin

Key Points

  • Kotlin productivity ke liye design ki gayi hai
  • Java me null safety nahi hoti
  • Android development me Kotlin preferred hai
  • Dono JVM par run karte hain

Interview One-Liner

Kotlin is concise, null-safe, and modern, while Java is verbose and prone to runtime null pointer errors.

Kotlin JVM, JS, Native

  • Kotlin JVM → Android apps, backend
  • Kotlin JS → Web apps
  • Kotlin Native → iOS, desktop apps

How Kotlin Works Internally

  1. You write Kotlin code
  2. Kotlin compiler converts it to Java bytecode
  3. JVM runs that bytecode

👉 That’s why Kotlin + Java work together.

7️⃣ Object Oriented Programming

  • OOP helps to organize code using real-world concepts.

Class & Object

Class
  • A blueprint for creating objects
  • Contains properties and functions
Object
  • An instance of a class
class Person(val name: String)
val p = Person("Nafis")

Constructors & init Block (Kotlin)

🔹 Constructors

Constructors are used to initialize objects in Kotlin.

✅ Primary Constructor

  • Written in the class header
  • Used to initialize properties
  • Main constructor of the class
class Student(val name: String, var age: Int)

val s = Student("Nafis", 20)

✅ Secondary Constructor

  • Used when extra initialization is required
  • Must call the primary constructor using this()
class Student(val name: String) {

    var age: Int = 0

    constructor(name: String, age: Int) : this(name) {
        this.age = age
    }
}

🔹 init Block

  • Runs automatically when an object is created
  • Used for initialization logic
  • Executes before the secondary constructor body
class User(val name: String) {

    init {
        println("User created: $name")
    }
}

🔹 Properties (Getter & Setter)

  • Kotlin automatically generates getter and setter
  • Custom getter/setter can be added
class Employee {
    var salary: Int = 0
        get() = field
        set(value) {
            field = value
        }
}

🔹 Visibility Modifiers

Control access level of class members.

Modifier Access
public Accessible everywhere (default)
private Only inside the class
protected Class and subclasses
internal Same module

Example

Q: What is the default visibility modifier in Kotlin?

🔹 Key Takeaways

  • Class is a blueprint, object is a real instance
  • Primary constructor is the main constructor
  • init block runs during object creation
  • Getters and setters are auto-generated
  • Visibility modifiers control access





8️⃣ Inheritance & Polymorphism in Kotlin

🔹 Inheritance

Inheritance allows one class to reuse another class’s code.

🔹 Polymorphism

Polymorphism allows the same function to behave differently based on the object type.

🔹 open Keyword

  • In Kotlin, classes and functions are final by default
  • To allow inheritance, the open keyword is required
open class Animal {
    fun eat() {
        println("Eating")
    }
}

❌ Without open, inheritance is not allowed.

🔹 Inheritance Syntax

  • Use : to inherit a class
  • Parent class constructor must be called using ()
class Dog : Animal() {
    fun bark() {
        println("Barking")
    }
}

🔹 override Keyword

  • Used to override methods or properties
  • Parent members must be marked open

Method Overriding 

open class Animal {
    open fun sound() {
        println("Animal sound")
    }
}

class Dog : Animal() {
    override fun sound() {
        println("Dog barks")
    }
}

🔹 Property Overriding

  • Parent property must be open
  • Child must use override
open class Vehicle {
    open val speed = 60
}

class Bike : Vehicle() {
    override val speed = 100
}

🔹 super Keyword

  • Used to call parent class methods or properties
open class Animal {
    open fun sound() {
        println("Animal sound")
    }
}

class Cat : Animal() {
    override fun sound() {
        super.sound()
        println("Cat meows")
    }
}

🔹 Polymorphism (Important)

  • Parent reference holding child object
  • Method call decided at runtime
val animal: Animal = Dog()
animal.sound()

Output

Dog barks

🔹 Abstract Classes

  • Cannot create objects
  • Used as base classes
  • Can have abstract and non-abstract methods
abstract class Shape {
    abstract fun area(): Int
}

class Square(val side: Int) : Shape() {
    override fun area(): Int {
        return side * side
    }
}

🔹 Interview Question

Q: Difference between open and abstract?

Answer:

  • open → Method has body, overriding is optional
  • abstract → No body, overriding is mandatory

🔹 Key Takeaways

  • Use open to allow inheritance
  • override replaces parent behavior
  • super calls parent implementation
  • Polymorphism provides flexibility
  • Abstract classes give structure





9️⃣ Interfaces

An interface is used to define what a class should do, not how it does it.

🔹 What is an Interface?

  • Interface contains method declarations
  • A class implements an interface
  • Supports multiple inheritance

🔹 Interface Methods

  • Methods are abstract by default
  • Interfaces do not have constructors
interface Vehicle {
    fun start()
    fun stop()
}

class Car : Vehicle {
    override fun start() {
        println("Car started")
    }

    override fun stop() {
        println("Car stopped")
    }
}

🔹 Default Implementations

  • Interfaces can have methods with body
  • Class may override or use default implementation
interface Printer {
    fun print() {
        println("Printing...")
    }
}

class HPPrinter : Printer

🔹 Multiple Inheritance via Interfaces

  • Kotlin does not support multiple class inheritance
  • Kotlin supports multiple interfaces
interface A {
    fun show()
}

interface B {
    fun show()
}

class Test : A, B {
    override fun show() {
        println("Implemented show")
    }
}

🔹 Resolving Conflict

If multiple interfaces have same method with implementation, conflict must be resolved explicitly.

interface A {
    fun display() {
        println("A")
    }
}

interface B {
    fun display() {
        println("B")
    }
}

class C : A, B {
    override fun display() {
        super<A>.display()
        super<B>.display()
    }
}

🔹 Interface vs Abstract Class

Interface Abstract Class
Multiple allowed Single inheritance
No state Can have state
No constructor Has constructor

🔹 Interview Question

Q: Can an interface have a method body in Kotlin?
Answer: Yes, using default implementation.

🔹 Key Takeaways

  • Interface defines behavior
  • Supports multiple inheritance
  • Default methods reduce boilerplate code
  • Conflicts resolved using super<InterfaceName>

🔟 Data Classes

Data classes are used to store data and help reduce boilerplate code.

🔹 What is a Data Class?

A special class designed to hold data.

Kotlin automatically generates the following:

  • toString()
  • equals()
  • hashCode()
  • copy()
  • componentN() functions

🔹 Syntax of Data Class 

data class User(val name: String, val age: Int)

val u1 = User("Nafis", 20)
println(u1)

Output

User(name=Nafis, age=20)

🔹 copy() Function

  • Creates a new object
  • Allows modification of selected values
val u2 = u1.copy(age = 21)
println(u2)

🔹 componentN() Functions

  • Used internally for destructuring
  • Generated for each constructor property
println(u1.component1()) // Nafis
println(u1.component2()) // 20

🔹 Destructuring Declarations

  • Extract object values into variables
val (name, age) = u1
println(name)
println(age)

🔹 Interview Question

Q: Why use a data class instead of a normal class?

Answer: Because it automatically generates useful methods and reduces boilerplate code.

🔹 Rules for Data Class

  • Must have at least one parameter
  • Primary constructor parameters must be val or var
  • Cannot be abstract, open, or sealed

🔹 Key Takeaways

  • Data class = data holder
  • copy() helps create modified objects
  • Destructuring makes code cleaner
  • Saves a lot of boilerplate code
