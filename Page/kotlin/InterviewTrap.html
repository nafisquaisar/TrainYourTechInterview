⚠️ Kotlin Interview Traps (Easy & Important)

These are very common interview traps in Kotlin.
Explained in simple language with real examples.

1️⃣ == vs ===

== (Value comparison)

  • Compares data/value

=== (Reference comparison)

  • Compares memory reference
val a = "Kotlin"
val b = "Kotlin"

println(a == b)   // true
println(a === b)  // may be false

👉 Use == in most cases.

2️⃣ lateinit vs lazy

lateinit

  • Used with var
  • Value set later
  • Cannot be null
  • Mostly used in Android
lateinit var name: String
name = "Nafis"

lazy

  • Used with val
  • Value created when first used
val data by lazy {
    10
}

👉 lateinit → Android views
👉 lazy → heavy objects

3️⃣ List vs MutableList

List

  • Read-only
val list = listOf(1, 2)

MutableList

  • Can add/remove data
val list = mutableListOf(1, 2)
list.add(3)

👉 List is not immutable, only read-only.

4️⃣ object vs companion object

object

  • Singleton
object AppConfig {
    val version = "1.0"
}

companion object

  • Works like static in Java
class User {
    companion object {
        fun create() = User()
    }
}

5️⃣ Flow vs LiveData

LiveData

  • Lifecycle aware
  • Used in UI

Flow

  • Coroutine based
  • Used in data layer
flow {
    emit(1)
}

6️⃣ val is NOT constant 

val list = mutableListOf(1)
list.add(2) // allowed

👉 val means reference cannot change, object can.

7️⃣ !! Operator Trap 

val name: String? = null
name!! // crash

👉 Avoid !!, use ?.

8️⃣ when is an expression 

val result = when (x) {
    1 -> "One"
    else -> "Other"
}

👉 Must return value.

9️⃣ Data class equality trap 

data class User(val id: Int)

val u1 = User(1)
val u2 = User(1)

println(u1 == u2) // true

🔟 Coroutine Scope Trap

❌ Wrong

GlobalScope.launch { }

✅ Correct

viewModelScope.launch { }

🧠 Interview Tip

Always say: "It depends on use case" and then explain.
