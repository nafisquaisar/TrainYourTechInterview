🔴 KOTLIN OBJECT & SPECIAL CLASSES

🔰 Introduction

Kotlin provides special class types to model restricted hierarchies, fixed sets of constants, and single-instance objects. These are heavily used in Android and backend development.

1️⃣7️⃣ Sealed Classes

Sealed classes represent a restricted class hierarchy. All subclasses must be defined in the same file.

sealed class Result

class Success(val data: String) : Result()
class Error(val message: String) : Result()

🔹 Sealed vs Enum

Sealed Class Enum Class
Can have different subclasses Fixed set of constants
Subclasses can have properties Limited flexibility
Supports inheritance No inheritance

🔹 Use Cases

  • API response handling
  • UI state management
  • Network result wrapper

🔹 Exhaustive when

when with sealed classes does not require else.

fun handle(result: Result) {
    when (result) {
        is Success -> println(result.data)
        is Error -> println(result.message)
    }
}

1️⃣8️⃣ Enum Classes

Enum classes define a fixed set of constant values.

enum class Direction {
    NORTH, SOUTH, EAST, WEST
}

🔹 Enum with Properties 

enum class Status(val code: Int) {
    SUCCESS(200),
    ERROR(500)
}

🔹 Enum with Methods 

enum class Operation {
    ADD {
        fun apply(a: Int, b: Int) = a + b
    },
    SUB {
        fun apply(a: Int, b: Int) = a - b
    }
}

1️⃣9️⃣ Object Keyword

The object keyword is used to create single instances or anonymous implementations.

🔹 Object Declaration

Creates a singleton object.

object AppConfig {
    const val VERSION = "1.0"
}

🔹 Singleton Pattern

Kotlin provides singleton behavior automatically using object.

object Database {
    fun connect() {
        println("Connected")
    }
}

🔹 Companion Object

Used to define static-like members inside a class.

class User {
    companion object {
        fun create(): User {
            return User()
        }
    }
}

🔹 Anonymous Objects

Used to create objects without a class.

val listener = object {
    fun onClick() {
        println("Clicked")
    }
}

✅ Summary

  • Sealed classes handle restricted hierarchies
  • Enums represent fixed constant values
  • Object keyword supports singleton and static behavior





🟣 KOTLIN FUNCTIONAL & GENERICS

🔰 Introduction

These concepts help you write reusable, type-safe, and high‑performance Kotlin code. They are very common in Android frameworks, libraries, and interviews.

2️⃣0️⃣ Generics

Generics allow you to write code that works with different data types while maintaining type safety.

🔹 Generic Classes

A generic class works with a type parameter.

class Box<T>(val value: T)

val intBox = Box(10)
val strBox = Box("Hello")

👉 T can represent any type.

🔹 Generic Functions

Functions that work with different types.

fun <T> printItem(item: T) {
    println(item)
}

🔹 Type Constraints

Restricts the type that a generic can accept.

fun <T : Number> sum(a: T, b: T): Double {
    return a.toDouble() + b.toDouble()
}

🔹 Variance (in, out)

Controls how types behave in inheritance.

  • out → Producer (read-only)
  • in → Consumer (write-only)
class Producer<out T>(val value: T)
class Consumer<in T> {
    fun consume(item: T) {}
}

🔹 Star Projections

Used when you don’t know the exact type.

fun printList(list: List<*>) {
    list.forEach { println(it) }
}

2️⃣1️⃣ Extensions

Extensions allow you to add functionality to a class without modifying it.

🔹 Extension Functions 

fun String.reverseText(): String {
    return this.reversed()
}

"Kotlin".reverseText()

🔹 Extension Properties 

val String.lastChar: Char
    get() = this[length - 1]

🔹 Limitations of Extensions

  • Cannot access private members
  • Do not actually modify the class
  • Resolved at compile time

2️⃣2️⃣ Inline, Reified & Tailrec

These features improve performance and readability.

🔹 Inline Functions

Inline functions copy function body at call site.

inline fun greet(action: () -> Unit) {
    action()
}

👉 Reduces overhead of lambda creation.

🔹 Reified Type Parameters

Allows access to type at runtime.

inline fun <reified T> checkType(value: Any) {
    if (value is T) println("Matched")
}

⚠️ Only works with inline functions.

🔹 Tail Recursion

Tail recursion optimizes recursive calls.

tailrec fun factorial(n: Int, result: Int = 1): Int {
    if (n == 0) return result
    return factorial(n - 1, result * n)
}

🔹 Performance Benefits

  • Inline reduces function call overhead
  • Reified avoids reflection
  • Tailrec prevents stack overflow

✅ Summary

  • Generics provide type safety and reusability
  • Extensions add functionality cleanly
  • Inline and tailrec improve performance
