🌐 Graph Data Structure – Complete Notes

1️⃣ What is a Graph?

A graph is a non-linear data structure used to represent relationships between objects.

  • A graph consists of vertices (nodes) and edges (connections)
  • It is used when data has complex relationships

👉 Real-life Examples

  • Social networks (users connected by friends)
  • Road maps and navigation
  • Computer networks
  • Web pages and links

2️⃣ Graph Structure (Visual Representation) 

      A ----- B
      | \     |
      |  \    |
      C ----- D

Explanation

  • A, B, C, D are vertices
  • Lines between nodes are edges
  • Graph can have cycles

3️⃣ Basic Graph Terminology

Term Description
Vertex A node in graph
Edge Connection between vertices
Degree Number of edges connected to a vertex
Path Sequence of vertices
Cycle Path that starts and ends at same vertex
Connected Graph Every vertex is reachable
Disconnected Graph Some vertices are isolated

4️⃣ Types of Graph

🔹 Undirected Graph

  • Edges have no direction

🔹 Directed Graph (Digraph)

  • Edges have direction

🔹 Weighted Graph

  • Edges have weights or cost

🔹 Unweighted Graph

  • All edges have same weight

🔹 Cyclic Graph

  • Contains cycles

🔹 Acyclic Graph

  • No cycles (DAG)

5️⃣ Graph Representation

🔹 Adjacency Matrix

  • Uses 2D array
  • Space: O(V²)

Example:

   A B C D
A  0 1 1 0
B  1 0 0 1
C  1 0 0 1
D  0 1 1 0

🔹 Adjacency List

  • Each vertex stores list of adjacent vertices
  • Space efficient

6️⃣ Adjacency List Implementation (Java) 

import java.util.*;

class Graph {
    int V;
    ArrayList<ArrayList<Integer>> adj;

    Graph(int V) {
        this.V = V;
        adj = new ArrayList<>();
        for (int i = 0; i < V; i++)
            adj.add(new ArrayList<>());
    }

    void addEdge(int u, int v) {
        adj.get(u).add(v);
        adj.get(v).add(u); // remove for directed graph
    }
}

7️⃣ Graph Traversals

🔹 Breadth First Search (BFS)

  • Breadth First Search (BFS) is a graph or tree traversal algorithm that explores nodes level by level. It starts from a given source node and visits all its immediate neighbors first, then moves to their neighbors, and continues this process until all reachable nodes are visited.

  • BFS uses a queue data structure to maintain the order of traversal.

  • Uses Queue

  • Level-wise traversal

void bfs(int start, ArrayList<ArrayList<Integer>> adj, int V) {
    boolean[] visited = new boolean[V];
    Queue<Integer> q = new LinkedList<>();

    visited[start] = true;
    q.add(start);

    while (!q.isEmpty()) {
        int node = q.poll();
        System.out.print(node + " ");

        for (int n : adj.get(node)) {
            if (!visited[n]) {
                visited[n] = true;
                q.add(n);
            }
        }
    }
}

🔹 Depth First Search (DFS)

  • Depth First Search (DFS) is a graph or tree traversal algorithm that explores a path as deep as possible before backtracking. It starts from a source node, visits one neighbor, then continues visiting the next unvisited neighbor until no more nodes are left on that path. After that, it backtracks and explores other paths.

  • DFS is commonly implemented using recursion or an explicit stack.

  • Uses Recursion / Stack

void dfs(int node, boolean[] visited, ArrayList<ArrayList<Integer>> adj) {
    visited[node] = true;
    System.out.print(node + " ");

    for (int n : adj.get(node)) {
        if (!visited[n]) {
            dfs(n, visited, adj);
        }
    }
}

🌳 Minimum Spanning Tree (MST)

A Minimum Spanning Tree is a subgraph of a connected, weighted, undirected graph that:

  • Connects all vertices
  • Has no cycles
  • Contains exactly V − 1 edges
  • Has the minimum possible total edge weight

Minimum Spanning Tree is a Spanning Tree in which sum of the weight of the edges is as minimum as possible.

Key Points (Interview Ready)

  • Graph must be connected
  • MST is unique if all edge weights are different
  • Used to minimize cost in network design

Applications

  • Road and network design
  • Electrical wiring
  • Cable and pipeline layout

🔹 Kruskal’s Algorithm

Kruskal’s algorithm is a greedy algorithm that builds the MST by selecting edges in increasing order of weight.

Steps

  1. Sort all edges by weight (ascending)
  2. Pick the smallest edge
  3. Add it to MST if it does not form a cycle
  4. Repeat until MST has V − 1 edges

Key Points

  • Works well for sparse graphs
  • Uses Disjoint Set (Union–Find) for cycle detection
  • Builds MST by edge selection

Time Complexity

  • O(E log E) due to sorting edges

When to Use

  • When edges are fewer
  • When edge list is easy to sort

🔹 Prim’s Algorithm

Prim’s algorithm is a greedy algorithm that builds the MST by expanding from a starting vertex.

Steps

  1. Start from any vertex
  2. Select the smallest edge connecting MST to a new vertex
  3. Add the vertex and edge to MST
  4. Repeat until all vertices are included

Key Points

  • Uses priority queue / min-heap
  • Expands one node at a time
  • Does not create cycles implicitly

Time Complexity

  • O(E log V) using min-heap
  • O(V²) using adjacency matrix

When to Use

  • When graph is dense
  • When adjacency matrix is used

🔸 Kruskal vs Prim (Quick Comparison)

Feature Kruskal Prim
Approach Edge based Vertex based
Cycle check Required (Union-Find) Not needed
Data structure Disjoint Set Priority Queue
Best for Sparse graph Dense graph

🎯 Interview One-Liners

  • MST: Connects all vertices with minimum total weight and no cycles
  • Kruskal: Picks smallest edges globally without forming cycles
  • Prim: Grows MST from a starting vertex
