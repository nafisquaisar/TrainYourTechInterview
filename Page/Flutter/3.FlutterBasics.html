Flutter Basics

1. Widgets (Stateless & Stateful)

Stateless Widget

Used when UI does not change after it is built.

class MyStateless extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text("I never change");
  }
}

Stateful Widget

Used when UI updates based on user interaction or changing data.

class MyStateful extends StatefulWidget {
  @override
  State<MyStateful> createState() => _MyStatefulState();
}

class _MyStatefulState extends State<MyStateful> {
  int count = 0;

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text("Count: $count"),
        ElevatedButton(
          onPressed: () => setState(() => count++),
          child: Text("Increase"),
        ),
      ],
    );
  }
}

2. Hot Reload & Hot Restart

Hot Reload

  • Updates UI without losing app state.
  • Best for UI changes.

Hot Restart

  • Fully restarts the app.
  • Resets all state and variables.
r  # hot reload
R  # hot restart

6. Layout Widgets

Row

Places widgets horizontally.

Row(
  mainAxisAlignment: MainAxisAlignment.center,
  children: [
    Text("A"),
    Text("B"),
  ],
);

Main Axis Alignment (Horizontal Alignment)

Controls how children are placed horizontally.

1. MainAxisAlignment.start

Left aligned.

mainAxisAlignment: MainAxisAlignment.start,

2. MainAxisAlignment.end

Right aligned.

mainAxisAlignment: MainAxisAlignment.end,

3. MainAxisAlignment.center

Centered horizontally.

mainAxisAlignment: MainAxisAlignment.center,

4. MainAxisAlignment.spaceBetween

First item at start, last at end, space only between.

mainAxisAlignment: MainAxisAlignment.spaceBetween,

5. MainAxisAlignment.spaceAround

Equal space around each child.

mainAxisAlignment: MainAxisAlignment.spaceAround,

6. MainAxisAlignment.spaceEvenly

Equal space between all children.

mainAxisAlignment: MainAxisAlignment.spaceEvenly,

Cross Axis Alignment (Vertical Alignment)

Controls vertical placement inside a Row.

1. CrossAxisAlignment.start

Align at the top.

crossAxisAlignment: CrossAxisAlignment.start,

2. CrossAxisAlignment.end

Align at the bottom.

crossAxisAlignment: CrossAxisAlignment.end,

3. CrossAxisAlignment.center

Center vertically.

crossAxisAlignment: CrossAxisAlignment.center,

4. CrossAxisAlignment.stretch

Widgets expand to fill height.

crossAxisAlignment: CrossAxisAlignment.stretch,

5. CrossAxisAlignment.baseline

Align based on text baseline.

Works only with text-based widgets.

crossAxisAlignment: CrossAxisAlignment.baseline,
textBaseline: TextBaseline.alphabetic,

Example Combining Both 

Row(
  mainAxisAlignment: MainAxisAlignment.spaceEvenly,
  crossAxisAlignment: CrossAxisAlignment.center,
  children: [
    Text("A", style: TextStyle(fontSize: 20)),
    Text("B", style: TextStyle(fontSize: 30)),
    Text("C", style: TextStyle(fontSize: 25)),
  ],
);

Expanded

Expands widget to fill available space.

Row(
  children: [
    Expanded(child: Container(color: Colors.red)),
    Expanded(child: Container(color: Colors.green)),
  ],
);

Properties of Expanded

1. child
  • Jo widget expand karna ho, wo yaha dete ho.
  • Container, Text, ListView, koi bhi widget ho sakta hai.
2. flex
  • Kitna proportion space lena hai, ye decide karta hai.
  • flex: 1 → normal space
  • flex: 2 → double space
  • Zyada flex value → zyada width/height lega.
3. fit
  • Default: FlexFit.tight
  • Tight: pura available space le lega.
  • Loose: jitna chahe utna space le, force nahi karta.

Example with all properties 

Row(
  children: [
    Expanded(
      flex: 1,
      fit: FlexFit.tight,
      child: Container(color: Colors.blue),
    ),
    Expanded(
      flex: 2,
      fit: FlexFit.loose,
      child: Container(color: Colors.orange),
    ),
  ],
);

# Flutter: Container + InkWell

A **Container** is used for styling, layout, spacing, and decoration.
An **InkWell** is used to make any widget tappable with ripple effects.

---

## Container Example

```dart
Container(
  padding: EdgeInsets.all(20),
  margin: EdgeInsets.all(10),
  color: Colors.blue,
  child: Text("Box"),
);

InkWell

InkWell allows tap interactions on any widget.
It provides ripple animation (material touch effect).

Why do we use InkWell?

  • To make any widget clickable
  • To detect gestures
  • To give ripple effect inside Material widgets
  • Used in buttons, cards, containers, list tiles, custom UI elements

Basic InkWell Example

InkWell(
  onTap: () {
    print("Tapped");
  },
  child: Container(
    padding: EdgeInsets.all(20),
    color: Colors.amber,
    child: Text("Tap Me"),
  ),
);

InkWell Functions

InkWell supports multiple gesture callbacks:

1. onTap

Triggered when user taps once.

onTap: () {},

2. onDoubleTap

Triggered on double tap.

onDoubleTap: () {},

3. onLongPress

Triggered when user presses and holds.

onLongPress: () {},

4. onTapDown

Finger touches the screen.

onTapDown: (details) {},

5. onTapUp

Finger leaves the screen.

onTapUp: (details) {},

6. onHover (Web/Desktop)

onHover: (value) {},

7. onHighlightChanged

Triggered when tap highlight changes.

onHighlightChanged: (value) {},

8. onSecondaryTap (Right click on desktop)

onSecondaryTap: () {},

Full Example With Multiple Gestures

InkWell(
  onTap: () => print("Tap"),
  onDoubleTap: () => print("Double Tap"),
  onLongPress: () => print("Long Press"),
  onTapDown: (_) => print("Tap Down"),
  onTapUp: (_) => print("Tap Up"),
  child: Container(
    padding: EdgeInsets.all(20),
    margin: EdgeInsets.all(10),
    decoration: BoxDecoration(
      color: Colors.green,
      borderRadius: BorderRadius.circular(10),
    ),
    child: Text("Gesture Box", style: TextStyle(color: Colors.white)),
  ),
);

When Should You Use InkWell?

  • Making custom buttons
  • Clickable containers or cards
  • Custom menu items
  • List tiles without default styling
  • When you want Material ripple effect

InkWell cannot show ripple unless inside a Material widget.
Use this pattern:

Material(
  color: Colors.transparent,
  child: InkWell(
    onTap: () {},
    child: Container(...),
  ),
);
